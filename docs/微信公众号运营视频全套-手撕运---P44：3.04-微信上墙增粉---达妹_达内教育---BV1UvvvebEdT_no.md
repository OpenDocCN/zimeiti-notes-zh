## ⚙️ 互动功能总览
以下是“嗨现场”后台主要的互动功能模块介绍，它们能帮助您丰富活动形式。

*   **活动推荐**：平台会推荐一些热门活动模板，如赛马、赛车、滚动抽奖、红包雨、3D签到等。
*   **基础展示功能**：
    *   **消息上墙**：开启后，用户发送的消息可以显示在大屏幕上。
    *   **弹幕**：允许用户的评论以弹幕形式飞过屏幕。
    *   **图片墙**：用户上传的图片可以组合成特定造型进行展示。
    *   **霸屏**：用户付费后，其发送的某条消息可以全屏展示。
*   **暖场启动**：包含签到墙（如3D签到）、活动倒计时等功能，用于活动开始前的氛围预热。
*   **抽奖活动**：提供多种抽奖方式，如滚动抽奖、红包雨、语音口令抽奖等。
*   **现场互动**：包含摇一摇、数钱、答题、拔河等竞技类游戏，支持多人同时与屏幕互动，活跃现场气氛。
*   **投票功能**：用于现场发起投票并实时统计结果。

## 🎯 核心功能详解：幸运抽奖设置
现场互动功能丰富，现在我们以“幸运抽奖”为例，讲解其基本设置流程。

### 创建抽奖活动
首先，在后台找到并点击“新建抽奖”。

![](img/4911093b86167986f6f1b7fa58063749_5.png)

![](img/4911093b86167986f6f1b7fa58063749_7.png)

### 选择抽奖类型
系统会提示选择抽奖方式，主要分为两类：
*   **导入名单抽奖**：适用于公司年会等内部活动，需提前导入参与者名单。
*   **扫码抽奖**：适用于公开活动，现场用户扫码即可参与。

**代码示例：选择抽奖类型**
（在实际平台界面中点击相应按钮即可，此处用伪代码表示逻辑）
```plaintext
if (活动类型 == “内部活动”) {
    选择“导入名单抽奖”；
} else {
    选择“扫码抽奖”； // 本次演示选择此项
}
```

### 设置奖项信息
选择“扫码抽奖”后，进入奖项设置页面。您需要为每个奖项配置以下信息：
1.  **奖品名称**：例如“一等奖”。
2.  **奖品实物**：例如“iPhone XS Max”。
3.  **奖品数量**：例如“1”。
4.  **每次抽取人数**：决定点击一次抽奖按钮抽出多少名获奖者。例如，若二等奖有2个名额，可设置为一次抽2人，或分两次各抽1人。
5.  **奖品图片**：上传奖品的展示图片，建议使用方形图片以获得最佳显示效果。

![](img/4911093b86167986f6f1b7fa58063749_9.png)

**公式：奖项总量与抽取次数的关系**
假设某个奖项的奖品总数量为 **N**，每次抽取人数设置为 **M**，则最多需要抽取 **ceil(N/M)** 次（ceil为向上取整函数）才能抽完该奖项。

![](img/4911093b86167986f6f1b7fa58063749_10.png)

![](img/4911093b86167986f6f1b7fa58063749_12.png)

### 配置参与与展示条件
奖项信息填写后，需注意以下设置：
*   **参与条件**：默认情况下，所有扫码用户均可参与。如需设置更复杂的条件（如按用户分组抽奖），通常需要开通付费高级功能。
*   **个性化界面**：平台提供多种抽奖视觉效果模板，部分免费，部分3D特效需要付费。
*   **关键规则**：系统通常默认“每人仅可中奖一次”，请在设置时留意此规则。

完成所有设置后，点击“保存”，该奖项即创建成功。

## ⚠️ 重要注意事项与操作技巧
成功创建奖项后，后台会显示奖项列表。在正式活动开始前，请务必注意以下操作：

1.  **测试与预览**：在活动前应进行抽奖测试。但测试抽出的奖项会被系统记录为已发出，可能导致正式活动时奖品数量不足。
2.  **使用“临时加奖”**：如果测试后奖品数量减少，可以使用“临时加奖”功能，快速补充奖品数量，以应对正式活动需求。
3.  **调整抽奖顺序**：系统默认展示最新创建或正在进行的奖项。合理的抽奖顺序通常是从三等奖开始，逐步过渡到一等奖。您可以通过编辑奖项或创建新的“测试奖品”来调整顺序，确保前台大屏幕按您设定的顺序展示抽奖环节。
