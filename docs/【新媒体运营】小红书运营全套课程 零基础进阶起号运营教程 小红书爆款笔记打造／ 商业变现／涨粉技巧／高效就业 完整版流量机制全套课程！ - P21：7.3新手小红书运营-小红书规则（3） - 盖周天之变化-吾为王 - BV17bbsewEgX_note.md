上一节我们介绍了小红书的基本规则框架，本节中我们来看看平台是如何决定将你的内容推送给哪些用户的。

## 平台推流规则概述

小红书平台并非随机分发内容。它会根据一套复杂的规则，对账号和笔记进行评估，从而决定流量的分配。这套规则的核心依据是**账号权重**。权重越高，获得初始推荐流量的机会和量级就越大。

那么，平台具体依据哪些维度来推送流量呢？其整体推流逻辑主要围绕“发现页流量”展开。

## 发现页流量的三重匹配机制

发现页是用户浏览内容的主要入口。平台在此处的推送极其精准，主要依赖以下三重标签进行匹配：

1.  **账号标签**
2.  **笔记标签**
3.  **话题与关键词标签**

以下是每一重标签的详细解析：

### 1. 账号标签

账号标签是平台对你账号的长期定位。它在**起号阶段**就已开始形成，并持续影响后续流量。设置账号标签主要通过：
*   **个人资料**：昵称、简介、头像。
*   **内容偏好**：你持续浏览、互动（点赞、收藏、评论）的笔记类型。
*   **发布内容**：你账号历史发布笔记的垂直领域。

**公式**：`账号标签 = 个人资料 + 浏览行为 + 发布历史`

### 2. 笔记标签

笔记标签决定了单篇内容会被推荐给哪类具体人群。平台通过分析笔记内容（文字、图片、视频）来判定其目标用户属性，例如：
*   **年龄层**：18-24岁，24-30岁，30-35岁等。
*   **消费层级**
*   **兴趣偏好**

**核心逻辑**：当笔记发布后，平台会先推送给一小部分可能感兴趣的用户（小范围测试）。如果这部分用户的**点击率、点赞、收藏、评论、转发**等互动数据良好，平台就会判定笔记标签准确，进而将笔记推送给更多同标签用户。

**代码逻辑示意**：
```python
if 笔记互动数据(点击率, 点赞等) > 平台阈值:
    扩大推荐范围(同标签用户)
else:
    停止推荐或减少推荐
```

### 3. 话题与关键词标签

这是最直接的流量匹配方式。平台会抓取你笔记中的：
*   **标题关键词**
*   **正文关键词**
*   **添加的话题**

当用户**搜索**这些关键词，或历史浏览行为表明其对这类关键词感兴趣时，你的笔记就更有可能被推荐给他们。

**总结**：平台通过`账号标签 -> 笔记标签 -> 关键词标签`这三重漏斗，层层筛选，力求将你的内容推送给最可能产生互动的精准用户。

## 标签的重要性与修正

精准的标签是获得流量的前提。如果标签混乱，会导致内容推送给错误的人群，即使平台给了初始流量（例如2000次曝光），也可能因为无人互动而数据惨淡。

**重要提示**：前期打标签务必严谨。一旦标签错误，修正需要花费大量时间：
*   **微调小标签**：约需7天。
*   **彻底更改大标签**：可能需15天或更久。

修正方法主要是通过**主动搜索目标关键词**并与之互动，来“告诉”平台你的新兴趣，但这需要一个学习周期。这也解释了为什么运营初期需要“养号”——通过持续浏览、互动同领域内容，来快速建立正确的账号标签。
