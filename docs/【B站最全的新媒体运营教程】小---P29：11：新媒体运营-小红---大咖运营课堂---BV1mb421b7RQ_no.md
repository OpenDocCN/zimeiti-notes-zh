## 用户标签判定机制 🔍

系统为用户打标签的主要依据是**用户浏览行为**。用户注册时选择的兴趣范围参考价值不大，它仅作为初期内容方向的引导，无法进入系统深层的用户标签池。

系统根据以下用户行为数据来判定其兴趣分类：

以下是判定用户标签的核心行为指标：

1.  **点击率**
    *   **公式**：`点击率 = (点击某类内容的次数) / (系统推送的该类内容总次数)`
    *   系统在发现页向用户推送多种内容（如美妆、家居、母婴）。用户点击哪类内容更多，系统就会判定其对该领域更感兴趣，并持续推送相关内容进行试探，从而打上强关联或弱关联的标签。

2.  **停留时长**
    *   用户点开笔记后的观看时长。如果用户对某类内容（如美妆）普遍停留时间短，说明具体内容不吸引他，系统会尝试推送该领域下的其他细分内容（如上妆、护肤）。
    *   如果用户对平时点击少的内容（如家居）某次停留很久，浏览仔细，系统可能判定其有潜在需求（如装修、购买特定商品），并相应调整推荐。

3.  **页面浏览量（PV）**
    *   用户浏览页面的深度。例如，用户不仅看了某篇笔记，还进入博主主页浏览了其他多篇笔记。提升用户在自己主页的浏览深度，对账号健康度有积极帮助。

4.  **互动行为**
    *   包括点赞、收藏、评论、关注。这些数据直接提升笔记的热度和权重，帮助笔记进入更大的推荐流量池，同时也能带动账号其他笔记和粉丝数的增长。

## 平台流量分发入口 🚪

了解用户标签后，我们来看小红书主要的流量分发入口。平台目前主要有四个流量入口：发现页、关注页、附近和购物。

以下是各流量入口的详细说明：

*   **发现页**
    *   这是笔记获取流量的主要来源。内容包含：
        1.  已关注达人发布的内容。
        2.  根据用户标签（点击率、兴趣等）匹配的推荐内容。
        3.  系统为丰富内容多样性而推送的“发散性内容”。
    *   简言之，发现页是系统基于算法进行个性化推荐的核心阵地。

*   **关注页**
    *   这里集中展示用户所有已关注博主的最新笔记。当关注列表很长时，用户可能会在此页寻找特定博主的内容。

*   **附近**
    *   基于手机定位，推送同城或附近的内容。此入口以具有强地域特征的内容为主，如美食、本地旅游、线下店推广等。
    *   **代码示例（逻辑描述）**：
        ```python
        if 用户打开“附近”入口：
            内容推荐 = 基于(用户地理位置) 筛选 (本地美食、景点、线下服务等笔记)
        ```
    *   对于运营线下实体店的账号，需要重点关注此入口的流量。

*   **购物**
    *   这是小红书向电商化发展的重要入口，直接匹配用户的购物需求。用户在此搜索商品或浏览商城。对于希望直接带货变现的账号，这是关键入口之一。
