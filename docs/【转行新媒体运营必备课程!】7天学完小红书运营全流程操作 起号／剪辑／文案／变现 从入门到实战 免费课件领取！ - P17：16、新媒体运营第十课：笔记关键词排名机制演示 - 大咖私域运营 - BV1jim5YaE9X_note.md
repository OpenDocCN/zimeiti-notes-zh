## 笔记发布后的系统处理流程

上一节我们介绍了搜索排名的影响因素，本节中我们来看看一篇笔记发布后，系统具体会经历哪些处理步骤。

### 第一步：内容审核

笔记发布后，系统首先会进行内容审核。审核的主要目的是判断笔记是否违规。

![](img/2daa3b59220e574bee645bf005bc3ea7_9.png)

以下是审核的主要关注点：
*   **敏感词**：检查笔记中是否包含平台禁止的敏感词汇。
*   **违规内容**：检查是否涉及平台限制准入的类目或其他非法信息。
*   **原创性**：检查笔记是否存在抄袭、素材高度重复或非原创等问题。

内容审核是笔记进入流量池的第一道门槛。

### 第二步：笔记收录

当内容审核通过后，笔记进入收录环节。笔记能否被收录，取决于两个核心因素：

**公式：笔记收录概率 = f(账号权重， 笔记质量)**

1.  **账号权重**：指账号在当前系统流量层级中的位置。权重高的账号通常有更高的初始曝光。
2.  **笔记质量**：这是决定收录的关键。笔记质量并非由系统主观判定，而是由**用户的行为数据**决定。

![](img/2daa3b59220e574bee645bf005bc3ea7_11.png)

例如：
*   一个高权重账号发布笔记，获得1万次初始曝光。如果点击量（“小眼睛”）不足10%（即少于1000），系统会判定该笔记质量不佳，可能不予收录。
*   一个低权重账号发布笔记，仅获得500次曝光，但点击量达到200次（点击率40%）。虽然绝对数值小，但用户互动积极，这篇笔记被收录的可能性反而更高。

![](img/2daa3b59220e574bee645bf005bc3ea7_12.png)

![](img/2daa3b59220e574bee645bf005bc3ea7_13.png)

**核心概念**：**用户互动数据（如点击率）是系统判断笔记质量的核心依据。**

### 第三步：特征提取与打标签

笔记被收录的同时，系统会对其内容进行分析，提取特征并打上标签。

系统会从多个维度提取信息，为笔记分类：
*   **用户行为反馈**：笔记分发给测试用户后，分析哪些标签的用户对其感兴趣。例如，一篇旅游笔记，若对“旅游”标签用户反馈良好，系统会强化其旅游相关标签。
*   **账号历史数据**：参考账号以往发布内容被打上的标签。
*   **内容关键词**：从笔记的标题、正文中直接提取核心关键词。

**代码示例（概念性描述）：**
```python
# 伪代码：系统提取笔记标签
def extract_note_tags(note_content, user_feedback, account_history):
    tags_from_keywords = analyze_keywords(note_content) # 从内容提取关键词
    tags_from_users = analyze_user_feedback(user_feedback) # 从用户行为分析标签
    tags_from_history = get_account_tags(account_history) # 获取账号历史标签
    final_tags = merge_and_rank_tags(tags_from_keywords, tags_from_users, tags_from_history) # 合并并排序标签
    return final_tags
```
最终，系统会为笔记打上如 `#云南`、`#旅游`、`#学生党`、`#避坑` 等一系列标签。这些标签决定了笔记将被归入哪些关键词的索引库中进行排序。

### 第四步：内容索引与动态排名

这是排名机制的核心。系统会根据提取的标签，在各个关键词的索引库中为笔记预先建立一个**初始排名**。

以下我们通过一个示例来演示这个过程：

假设我们发布一篇笔记，标题为：**《8月去云南旅游的完整攻略，学生党必看避坑指南》**。

系统可能提取出 `云南`、`旅游`、`云南旅游`、`学生党`、`避坑` 等标签。随后，笔记会进入这些标签对应的“索引库”中，并拥有一个初始排名位置。

![索引库排名示意图](img/2daa3b59220e574bee645bf005bc3ea7_11.png)

![](img/2daa3b59220e574bee645bf005bc3ea7_15.png)

*   在“云南”关键词下，笔记初始排名可能是第7位（G）。
*   在“旅游”这个更宽泛的关键词下，排名可能是第6位（F）。
*   在“云南旅游”这个核心关键词下，排名可能是第4位（D）。

![](img/2daa3b59220e574bee645bf005bc3ea7_16.png)

**排名的动态调整：**
初始排名并非一成不变。系统会根据笔记在**搜索场景下**的用户行为数据（如上节课提到的搜索点击率、互动率等）进行动态调整。
*   **排名上升**：如果在“云南旅游”下，笔记的点击和互动数据很好，其在该关键词下的排名可能会从第4位（D）上升到第2位（B）。
*   **排名下降或移除**：如果在“学生党”标签下，长期没有相关搜索用户点击，笔记可能会从这个关键词的排名中被移除。在“旅游”大词下，由于竞争激烈且不够精准，排名可能会缓慢下降。

**核心概念**：**关键词排名是一个基于实时用户反馈的动态竞争过程。**

![](img/2daa3b59220e574bee645bf005bc3ea7_18.png)

### 第五步：动态排名与流量分配

现代新媒体平台还有一个更智能的机制：**动态排名（Dynamic Ranking）**。

![](img/2daa3b59220e574bee645bf005bc3ea7_20.png)

![](img/2daa3b59220e574bee645bf005bc3ea7_21.png)

![](img/2daa3b59220e574bee645bf005bc3ea7_22.png)

平台为了打破内容同质化，将流量更公平地分发给不同创作者，不会为每个关键词固定一个唯一的排名列表。

![](img/2daa3b59220e574bee645bf005bc3ea7_23.png)

![](img/2daa3b59220e574bee645bf005bc3ea7_24.png)

![动态排名示意图](img/2daa3b59220e574bee645bf005bc3ea7_20.png)

如图所示，对于同一个搜索词“云南旅游”：
*   用户A搜索时，看到的排名顺序可能是：笔记1 -> 笔记2 -> 笔记C...
*   用户B搜索时，看到的排名顺序可能变为：笔记A -> 笔记B -> 笔记C...

系统会根据搜索用户的个人画像（历史喜好、互动行为等），微调排名结果，以实现更精准的个性化推荐，同时让更多优质笔记有机会获得曝光。

## 总结

本节课我们一起学习了新媒体平台搜索排名机制的完整流程：
1.  **内容审核**：筛查违规，确保内容安全。
2.  **笔记收录**：由**账号权重**和**用户数据驱动的笔记质量**共同决定。
3.  **特征提取**：系统分析内容、用户反馈，为笔记打上多维标签。
4.  **内容索引**：笔记根据标签进入各关键词索引库，获得**初始排名**。
5.  **动态调整**：排名根据搜索场景下的**实时用户行为数据**（点击率、互动等）持续波动。
6.  **动态展示**：最终搜索结果会结合个性化推荐，进行**动态排名**，以优化用户体验和平台生态。

![](img/2daa3b59220e574bee645bf005bc3ea7_26.png)

理解这套机制，有助于我们更有针对性地优化笔记内容和关键词策略，从而提升在搜索流量中的竞争力。
