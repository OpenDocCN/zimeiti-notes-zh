在本节课中，我们将学习亚马逊运营中两个核心且实用的高级技巧：如何通过模板文件批量上传商品，以及如何通过合并与拆分变体来管理产品列表和优化评价。这些操作是高效管理多SKU产品的关键。

## 模板上传步骤详解

上一节我们介绍了单个商品上传的基础知识，本节中我们来看看如何通过模板文件批量上传商品，这对于管理多属性（变体）产品至关重要。

![](img/3817aa00a82fdb4938511baba4347d20_9.png)

### 第一步：下载商品模板

![](img/3817aa00a82fdb4938511baba4347d20_11.png)

![](img/3817aa00a82fdb4938511baba4347d20_13.png)

首先，我们需要从卖家后台下载对应类目的商品上传模板。

![](img/3817aa00a82fdb4938511baba4347d20_15.png)

1.  登录亚马逊卖家后台，进入“目录” > “添加商品”页面。
2.  在添加商品页面，选择“**我正在上传文件来添加多个商品**”选项。
3.  点击“生成模板”或类似按钮，进入模板下载页面。

![](img/3817aa00a82fdb4938511baba4347d20_17.png)

![](img/3817aa00a82fdb4938511baba4347d20_19.png)

### 第二步：筛选并确定产品类目

为了下载正确的模板，我们必须确定产品的精确类目。以下是操作方法：

![](img/3817aa00a82fdb4938511baba4347d20_21.png)

1.  前往亚马逊前台（买家页面），找到与你产品类似的商品。
2.  在商品详情页的“产品信息”部分，找到并记录下完整的类目路径（例如：美妆 > 工具 > 化妆工具 > 假睫毛）。
3.  返回卖家后台的模板下载页面，按照记录的类目路径，一级一级地筛选，直到找到最末端的准确类目。
4.  选择该类目，并生成模板文件进行下载。

### 第三步：填写模板信息

![](img/3817aa00a82fdb4938511baba4347d20_23.png)

![](img/3817aa00a82fdb4938511baba4347d20_24.png)

下载模板后，用Excel或类似表格软件打开。模板中通常有大量列，我们需要重点关注以下核心部分：

*   **商品类型和SKU**：这是识别商品的关键。
    *   **父体SKU**：代表整个变体系列，例如 `LASHES`。
    *   **子体SKU**：代表具体的变体，通常由父体SKU加上变体属性构成，例如 `LASHES-BLACK-15MM`、`LASHES-WHITE-12MM`。
*   **品牌信息**：如果尚未完成品牌备案，可以使用通用品牌名，如 `NP`（No Brand）或 `Generic`。若计划注册品牌，可预先在此处填写计划注册的品牌名，便于日后更新。
*   **商品编码**：如UPC、EAN等。**注意：父体商品通常不需要填写商品编码**，只需为每个子体填写即可。
*   **关键属性**：包括商品标题、制造商、重量等。标红字段通常为必填项，但**并非所有标红项都必须填写**，这取决于具体类目，需要根据上传后的报错提示或经验判断。
*   **图片链接**：需要填写主图和辅图（最多9张）的URL地址。图片URL可以通过图床工具生成，或从其他电商平台（如1688）的商品图片链接中获取。**图片字段并非强制必填**，可以后台上传。
*   **变体关系**：这是模板上传的核心，用于建立父子变体关系。
    *   在 `Relationship Type` 列，父体选择 `Parent`，子体选择 `Child`。
    *   在 `Parent SKU` 列，所有子体都需要填写其所属父体的SKU（例如 `LASHES`），父体此行留空。
    *   在 `Variation Theme` 列，定义变体主题，如 `Color-Size`（颜色-尺寸）。
    *   在对应的变体属性列（如 `Color Name`， `Size Name`），为每个子体填写具体的属性值（如 `Black`， `15MM`）。**所有子体的属性值必须完全一致**，包括大小写。
*   **更新与删除操作**：`Update Delete` 列至关重要，它决定模板执行的操作。
    *   **Update**：用于**从无到有**创建新商品或新变体。
    *   **Partial Update**：用于**部分更新**已存在商品的信息（如修改标题、价格、增加变体）。**警告**：在已有父体下新增子体时，父体必须选择 `Partial Update`，新增的子体选择 `Update`。若父体误选 `Update`，会导致原有其他子体丢失。
    *   **Delete**：用于删除商品或变体。
*   **商品描述与关键词**：填写商品要点（Bullet Points）、商品描述（Description）和搜索关键词（Keywords）。可以利用表格的批量填充功能高效填写多个变体。

### 第四步：上传并检查模板

填写完毕后，保存文件。返回卖家后台的“上传文件”页面，上传填写好的模板文件。系统会进行处理，并生成处理报告。务必仔细查看报告，处理任何错误或警告信息。

## 变体合并与拆分实战

掌握了模板上传后，我们来看一个高级应用场景：通过合并与拆分变体来优化产品列表，特别是处理差评。

![](img/3817aa00a82fdb4938511baba4347d20_35.png)

### 为何要合并/拆分变体？

![](img/3817aa00a82fdb4938511baba4347d20_40.png)

![](img/3817aa00a82fdb4938511baba4347d20_41.png)

主要目的是进行**差评隔离**。当一个热销变体（例如“黑色-15mm”）收到大量差评，拉低整个产品列表的星级评分时，我们可以将该差评变体从主链接中分离出去，从而保护主链接的评分。

![](img/3817aa00a82fdb4938511baba4347d20_42.png)

### 操作核心：分两步走

所有变体操作都遵循“**先打散，后重组**”的原则，并通过模板操作以保证有记录可循。

![](img/3817aa00a82fdb4938511baba4347d20_44.png)

**第一步：打散变体（删除父体）**

![](img/3817aa00a82fdb4938511baba4347d20_46.png)

![](img/3817aa00a82fdb4938511baba4347d20_47.png)

![](img/3817aa00a82fdb4938511baba4347d20_49.png)

目的是解除现有的父子变体关系，让所有子体变为独立的商品。

1.  新建一个模板，只保留需要打散的变体系列的**父体SKU**一行。
2.  在该行的 `Update Delete` 列填写 **`Delete`**。
3.  其他所有列（如变体关系、子体信息）留空或删除。
4.  上传该模板。操作成功后，原父体被删除，所有子体变为独立Listing，但**商品评价会跟随各自的子体**。

![](img/3817aa00a82fdb4938511baba4347d20_51.png)

**第二步：重组变体（合并或新建）**

![](img/3817aa00a82fdb4938511baba4347d20_53.png)

根据目标，重新创建父子关系。

![](img/3817aa00a82fdb4938511baba4347d20_55.png)

![](img/3817aa00a82fdb4938511baba4347d20_56.png)

*   **场景A：合并两个独立Listing**
    1.  将两个独立Listing（假设为A和B）的所有子体信息填入新模板。
    2.  创建一个**新的父体SKU**（不能与已删除的旧父体SKU相同）。
    3.  在新模板中建立新的父子变体关系：新父体行选择 `Parent` 和 `Update`；所有子体行选择 `Child`，并在 `Parent SKU` 列填写新的父体SKU，`Update Delete` 列选择 `Partial Update`（因为子体是已存在的商品）。
    4.  上传模板，即可将A和B合并到一个新的变体组下。
*   **场景B：移除差评变体后重组**
    1.  在第一步打散变体后，差评变体（如“黑色-15mm”）已成为独立Listing。
    2.  我们可以直接在后台上传模板，**单独删除**这个差评变体（使用 `Delete`）。
    3.  然后，使用新模板，将剩下的优质子体（如“白色-15mm”、“黑色-12mm”、“白色-12mm”）合并到一个新的父体下（操作同场景A）。
    4.  这样，差评随原变体被移除，主链接评分得到提升。被移除的库存可以后期通过新建链接或重新贴标等方式处理。

![](img/3817aa00a82fdb4938511baba4347d20_58.png)

![](img/3817aa00a82fdb4938511baba4347d20_60.png)

### 重要注意事项

![](img/3817aa00a82fdb4938511baba4347d20_62.png)

![](img/3817aa00a82fdb4938511baba4347d20_64.png)

1.  **模板操作优先**：所有合并、拆分、删除操作都建议通过模板完成，因为后台直接操作无记录，一旦误删难以恢复。模板操作有编号，可联系亚马逊客服追溯。
2.  **谨慎操作**：频繁合并拆分变体可能被亚马逊系统判定为滥用，导致店铺受到警告或限流。此方法应作为处理紧急差评的补救措施，而非常规操作。
3.  **属性一致性**：合并变体时，确保要合并的子体具有相同或兼容的变体主题（如都是`Color-Size`）。
