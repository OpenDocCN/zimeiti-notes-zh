在本节课中，我们将要学习小红书店铺运营的前期整体规划。店铺规划是运营的基础，良好的开端能为后续的流量和转化打下坚实基础。本节将重点讲解前期需要准备的商品、数据以及操作人员规划。

上一节我们介绍了店铺运营的整体框架，本节中我们来看看具体的前期准备工作该如何进行。

## 前期规划概述

店铺运营的前期规划，核心目标是建立店铺的基础数据量。只有保证了基础数据量，店铺才能获得自然的成交比例。前期规划主要包含三个部分：商品准备、基础数据准备和操作人员准备。

## 商品准备 🛒

商品是店铺的核心。以下是关于商品数量与发布的具体要求。

商品整体数量需控制在 **7~45个** 之间。即使是运营单品单店，也应保证在此数量区间内。具体构成如下：
*   **主产品**：1件。这是我们主要推广的核心产品。
*   **副产品**：2~4件。这些是预备打造为第二、第三爆款的产品。
*   **补量产品**：5~10件。这些产品的主要作用是丰富店铺SKU，增加用户选择，提升店铺整体活跃度和账号基础权重，无需额外进行数据维护。

产品发布并非一次性完成，而是有技巧地分批上架。

以下是产品发布的具体技巧：
1.  **先上副产品**：首先上架1-2件副产品，主产品暂不上架。
2.  **延迟上主品**：主产品通常在第二天、第三天或第四天发布。这样做是为了先测试账号权重，让小红书系统先识别并录入我们的店铺和副产品，完成基础的排名划分。
3.  **每日上架限制**：每天上架的产品数量建议不超过3件，以1-2件为佳。
4.  **把握上架时间**：
    *   副产品应避开行业搜索高峰期上架，避免与主产品竞争流量。
    *   主产品应在行业高峰期**前30分钟**上架。这能给系统约5-10分钟的录入时间，使产品在高峰期来临时能获得更靠前的排名，从而得到更多曝光。

行业高峰期通常有2-3个，选择其中一个操作即可。一般而言，下午的高峰期流量较大。若经营办公用品等类目，午间高峰期也可能表现良好。大类目可选下午，小类目可选中午或晚上，但晚上时段的整体成交量通常不高。

**核心要点**：店铺前期运营的商品数量**下限是7件**。低于7件，店铺权重难以提升。单店可准备7件，多店模式建议接近15件，后续可逐步补充。

## 基础数据准备 📊

商品上架后，需要为其打造基础数据，以提升商品竞争力与系统评分。

基础数据主要包括评价、收藏加购和销量。
*   **评价**：每个商品前期至少需要准备**5个**评价，才能支撑起正常的销售转化。
*   **收藏加购**：目标数量应在**50个以上**。虽然没有硬性规定，但数量越多，系统对商品的评分越高，越有利于后期的营销推广。
*   **销量**：
    *   **大类目**：需要**20个以上**的基础销量。
    *   **小类目**：需要**10个以上**的基础销量。
    *   **冷门类目**：需根据产品具体价格和特性单独分析，无统一标准。

## 操作人员准备 👥

操作人员负责执行前期的数据优化工作，其操作必须精准对标目标人群。

操作的核心是对对标店铺进行深入分析，并据此操作。需要分析的数据维度包括：
1.  **产品分析**：价格区间、产品类型、时效性。
2.  **人群分析**：性别、年龄、购买习惯、消费价格敏感度与类型。

操作人员需要根据以上分析得出的“人群标签”，来执行店铺的基础数据操作（如做销量、收藏等）。目的是在店铺前期就确立精准的**人群画像和基础属性**。

**为什么必须在前中期做好人群规划？**
如果前期人群混乱，后期当店铺数据量增长后再想修正，难度会成倍增加。前期规划好人群标签，即使后期参与平台大型引流活动，对店铺的精准流量影响也较小。反之，若人群未固定就参与大型活动，可能导致流量泛化，活动结束后店铺失去精准流量，只能依赖活动生存。

因此，前期通过操作人员进行的对标优化，是确保店铺长期健康运营的关键。

## 核心数据总结

![](img/4a09e9fb377d953b96f9efc0caabaf3e_9.png)

为确保清晰，此处再次强调两个最基础的数据要求：
*   **主产品基础销量公式**：
    *   大类目：`销量 >= 20`
    *   小类目：`销量 >= 10`
*   **店铺商品数量公式**：`7 <= 商品总数 <= 45`（前期最低7件）
