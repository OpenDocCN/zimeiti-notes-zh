# 亚马逊旺季促销设置教程：P1：Prime折扣与优惠券详解

在本节课中，我们将学习亚马逊Prime折扣和优惠券的核心概念、设置方法以及关键的叠加规则。掌握这些知识，能帮助你在旺季（如秋季促销）来临前，有效设置促销活动，避免因设置错误导致重大损失。

## 亚马逊秋季促销与旺季背景

亚马逊秋季促销将于太平洋夏季时间10月8日凌晨12点开启，持续两天。10月作为第四季度的首月，标志着销售旺季正式到来，广告竞价和销量通常都会开始上升。对于许多卖家而言，这是年度重要的盈利时期。

## Prime会员专享折扣详解

上一节我们介绍了旺季背景，本节中我们来看看Prime会员专享折扣。这个折扣是面向亚马逊Prime会员显示的专属价格。

*   **定义**：Prime会员在搜索页或商品详情页能看到此价格。折扣幅度**至少为原价的10%**，最高可达**原价的80%**（即打两折）。
*   **费用**：消费者可直接以此价格购买，**亚马逊不向卖家收取额外费用**。
*   **设置建议**：建议所有符合条件的卖家都设置。在大促期间（如秋季促销），平台可能会要求**最低折扣15%** 并收取**50美金**的提报费用，但同时能获得专属流量和标识等福利。
*   **限制条件**：
    1.  Prime折扣价必须**低于最近30天的最低售价**。
    2.  会影响后续秒杀价格的设定。
    3.  商品需为新品，非成人用品，拥有三星以上评级或无评级。
*   **适用场景**：适用于价格调整空间不大的常规产品。若无法满足“低于30天最低价”的条件，则应考虑使用优惠券。

## 优惠券设置与计算

了解了Prime折扣后，我们来看看功能更灵活的促销工具——优惠券。

*   **定义**：优惠券允许设置**5%至50%** 的折扣或固定金额折扣。
*   **玩法多样性**：可以针对单个ASIN设置，为“订购省”买家提供折扣，或设置“重新下单优惠券”吸引老顾客。
*   **核心成本**：每次优惠券被兑换，卖家需支付 **$0.60的兑换费**。
*   **预算计算**：优惠券预算的计算方式较为特殊。预算消耗包括“给消费者的优惠金额”和“兑换费”。
    *   **计算公式**：`单次兑换消耗预算 = 优惠金额 + $0.60`
    *   **举例说明**：商品原价$20，设置$5优惠券。消费者兑换购买后，消耗的预算为 `$5（优惠）+ $0.60（费用） = $5.60`。因此，$100的预算大约可支持 `$100 / $5.60 ≈ 17` 次兑换，而非 `$100 / $0.60`。

## 至关重要的折扣叠加规则

介绍完两种主要促销工具，本节我们来探讨最关键也是最危险的部分：折扣叠加规则。理解错误可能导致商品被“0元购”。

*   **核心规则**：**优惠券可以与任何其他促销活动叠加**，包括秒杀、Prime折扣、企业价格、站外促销等。
*   **叠加计算逻辑**：所有计算都是在现有折扣价的基础上，再叠加优惠券的折扣。
    *   **代码示例（逻辑）**：
        ```python
        # 假设商品原价 original_price = $100
        # 秒杀折扣为30%，则秒杀价 promotion_price = original_price * (1 - 0.30) = $70
        # 同时存在一个50%的优惠券，则最终价 final_price = promotion_price * (1 - 0.50) = $35
        ```
*   **风险与应对**：在使用优惠券时，必须仔细计算最终的成交价。最直接的避险方法是：**在进行大幅度的促销（如7天秒杀）时，暂停使用优惠券**。
*   **重要提示**：
    1.  优惠券的**停用并非即时生效**。点击停用后，已领取优惠券的买家仍有约**半小时**的使用时间。
    2.  优惠券的**生效也非即时**，创建后可能需要**数小时（官方称最快6小时）** 才能在前台显示。
