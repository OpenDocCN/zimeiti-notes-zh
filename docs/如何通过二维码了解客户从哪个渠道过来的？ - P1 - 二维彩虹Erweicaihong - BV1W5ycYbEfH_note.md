# 渠道追踪教程：P1：如何区分客户来源渠道

在本节课中，我们将学习如何通过二维码和监测链接来精确追踪客户是从哪个运营平台或哪位运营人员引流而来的。这对于多平台、多账号运营的公司至关重要。

## 概述

许多公司在进行多平台运营时，面临一个共同问题：用户通过不同平台和账号添加至微信后，无法区分其具体来源。本节课程将介绍两种简单有效的方法来解决这个问题，核心是使用**渠道来源统计**功能。

## 方法一：使用监测链接（UTM参数）

如果你拥有自己的网站或统计后台，推荐使用监测链接（或包含监测链接的二维码）进行追踪。这种方法可以记录用户点击后的行为数据。

![](img/e093bfb72cd70dd1e57c87aa42dd8c40_1.png)

上一节我们概述了问题，本节中我们来看看第一种解决方案的具体操作。

![](img/e093bfb72cd70dd1e57c87aa42dd8c40_2.png)

1.  **访问UTM设置**：在二维彩虹后台，找到并点击“UTM”按钮。
    
    ![](img/e093bfb72cd70dd1e57c87aa42dd8c40_1.png)
    ![](img/e093bfb72cd70dd1e57c87aa42dd8c40_2.png)

![](img/e093bfb72cd70dd1e57c87aa42dd8c40_4.png)

2.  **自定义参数**：在UTM设置中，你可以完全自定义链接参数来标识不同来源。例如：
    *   `utm_source=渠道一`
    *   `utm_medium=运营人员_刘亦菲`
    *   `utm_campaign=百度推广`
    *   你还可以添加自定义编号（如`id=123456`）来代表渠道或产品编号。
    
    ![](img/e093bfb72cd70dd1e57c87aa42dd8c40_4.png)

3.  **生成与使用**：生成带有这些参数的监测链接或二维码。所有通过此链接/二维码的访问数据（如点击次数、用户停留时间等）都会在你的网站统计后台（如Google Analytics）中显示，而非二维彩虹后台。

通过这种方式，你可以清晰地区分“渠道一”、“渠道二”等各自带来了多少扫码添加。

## 方法二：使用独立命名的二维码

对于没有自己网站统计后台的小型商家或团队，可以采用更直接的方法：为每位运营人员或每个渠道生成独立的二维码。

了解了需要自有后台的监测链接方法后，我们来看看更轻量级的替代方案。

以下是操作步骤：

1.  **生成独立二维码**：为每位运营人员或每个推广渠道生成一个单独的二维码。
2.  **重命名二维码**：在二维彩虹后台对每个二维码进行重命名，例如“运营人员_张三_微博渠道”。
3.  **查看扫描数据**：在后台可以查看每个被重命名的二维码的扫描统计，数据包括：
    *   **扫描时间与地点**
    *   **扫描设备**（iOS、安卓、桌面浏览器）
    *   **精确到城市**（如北京、广州、深圳）
    *   **扫描总数**与**唯一扫描数**（区分一人扫一次还是一人扫多次）

    ![](img/e093bfb72cd70dd1e57c87aa42dd8c40_6.png)

4.  **分渠道查看**：后台数据可以按渠道（如KTV渠道、微博渠道、官网链接）分类查看，方便对比各渠道效果。

## 两种方法的区别

现在我们已经了解了两种方法的具体操作，接下来总结一下它们的核心区别，以帮助你做出选择。

*   **方法一（监测链接）**：可以追踪用户**点击链接后**的行为（如是否完成注册、停留了多久）。你不知道的只是他们是否仅仅点击而未产生后续行为。
*   **方法二（独立二维码）**：只能统计到**有多少人扫码**以及扫码的基本信息，但无法知道用户扫码访问后是否继续操作或立即退出。

简单来说，方法一提供了“转化过程”的深度洞察，而方法二仅提供“访问入口”的广度计数。

## 总结

本节课中我们一起学习了两种区分客户来源的方法：
1.  通过**自定义UTM参数的监测链接**，适用于有自有统计后台的用户，能进行深度行为追踪。
2.  通过**为不同渠道生成并重命名独立二维码**，适用于所有用户，能快速统计各渠道的扫码量。

![](img/e093bfb72cd70dd1e57c87aa42dd8c40_6.png)

你可以根据自身是否具备网站后台这一条件，选择最适合的方案来精确追踪每一位客户的来源渠道。
