上一节我们介绍了系统如何判定内容标签，本节中我们来看看平台如何判定用户标签。这个过程能帮助我们反推笔记被推荐给用户的逻辑，从而让我们的内容更精准地触达目标受众。

## 用户标签判定机制

系统为用户打标签的主要依据是用户的**浏览行为**。用户注册时选择的兴趣范围参考价值不大，仅作为初期内容方向的引导，无法进入系统深层的用户标签池。

系统通过分析用户的具体互动行为来判定其兴趣分类。以下是系统评估用户偏好的几个核心行为指标：

以下是判定用户标签的几个关键行为指标：

1.  **点击率**
    *   **公式**：`点击率 = (点击某类内容的次数) / (系统推送的该类内容总次数)`
    *   系统在发现页向用户推送多种内容（如美妆、家居、母婴）。用户点击哪一类内容更多，系统就能计算出其偏好。例如，用户点击了10条推送，其中美妆类3次，家居类2次，母婴类0次，系统会判定该用户对美妆更感兴趣，对母婴不感兴趣，并持续推送更多美妆内容进行试探。

2.  **停留时长**
    *   用户点开笔记后的观看时长。如果用户频繁点击某类内容但很快划走，说明具体内容不吸引他。系统会尝试推送该大类下的其他细分内容（如美妆大类下的“上粉”或“护肤”）。反之，如果用户对某条笔记（如家居内容）停留很久，仔细阅读图片和评论，系统可能判断他有相关需求（如装修、购买特定商品），并相应调整推荐。

3.  **页面浏览量**
    *   用户浏览页面的深度。例如，用户不仅看了一篇笔记，还点进博主主页，并连续看了第二、第三篇笔记。这个数值（PV）对账号健康度有积极影响。在粉丝基数较小时，提升用户在你主页的浏览深度，能为账号带来非常正面的评分。

4.  **互动行为**
    *   包括点赞、收藏、评论、关注。这些数据直接关系到笔记能否进入更大的推荐流量池。数据越好，笔记获得推荐的机会越高，同时也能带动账号内其他笔记的流量，并有助于粉丝增长和后续变现。

简单来说，系统通过上述行为，为用户构建一个动态的“兴趣云图”。例如，一个用户的标签云中，“美妆”这个词最大，代表他是美妆内容的深度爱好者，系统便会主要推送相关内容。平台、用户和博主（内容创作者）三者在此关联：博主提供有价值的内容给用户，平台则负责将合适的内容匹配给用户，三方共同维系平台的生态。

## 平台流量分发入口

了解用户标签后，我们来看看小红书主要的流量分发入口。作为用户你可能经常使用，但需要分析其区别以指导运营。

小红书目前主要有四个流量入口：

以下是四个主要的流量入口及其特点：

1.  **发现页**
    *   这是笔记和账号获取流量的**主要来源**。包含三部分内容：
        *   **关注达人**：你已关注的博主发布的内容。
        *   **兴趣内容匹配**：系统根据你的用户标签（点击率、停留时长等）推荐的、你可能喜欢的内容。
        *   **发散性内容匹配**：系统为了保持平台内容多样化，偶尔推荐你可能未接触过但其他用户喜欢的内容。

2.  **关注页**
    *   这里**仅展示**你已关注的所有博主发布的笔记。用户通常从这里直接查看强关联博主的更新。

3.  **附近页**
    *   根据手机定位，推送附近及同城的相关内容。**强地域特征**的内容（如美食、本地旅游、线下店推广）在此板块推荐权重较高。如果你是线下实体店（如美容院、餐厅），在做信息流广告时需要重点关注附近人群的推送。

4.  **购物页**
    *   这是小红书向电商发展的重要入口，直接匹配用户的购物需求。目前用户习惯仍在培养中，许多人仍倾向于先搜索笔记（种草），而非直接进入商城购物。该入口与直播带货、商品笔记等内容紧密相关。
