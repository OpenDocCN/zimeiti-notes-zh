# 亚马逊跟卖软件教程：P1：软件核心功能与数据源解析 🔍

在本节课中，我们将要学习一款亚马逊跟卖软件的核心工作原理。我们将重点了解其数据来源的广泛性以及如何利用筛选条件找到你需要的产品。理解这些是有效使用该软件的基础。

上一节我们介绍了课程主题，本节中我们来看看该软件的数据来源基础。

![](img/1f40781e5feda1ab95e61276e7c80a21_1.png)

## 数据来源：亚马逊实时数据

![](img/1f40781e5feda1ab95e61276e7c80a21_3.png)

该软件不局限于任何特定形式的跟卖。它抓取的是亚马逊的实时数据。不要用狭隘的眼光去看任何一个已经成型的运营模式，每种模式都有其优势。

![](img/1f40781e5feda1ab95e61276e7c80a21_4.png)

软件的功能实现，完全依赖于其数据源。无论你计划如何选择跟卖的产品，最终都是从亚马逊平台上进行选择的。

我们软件是对接的亚马逊API接口，采集的是亚马逊前台的实时数据。因此，软件采集出来的数据是全面且实时的。你可以根据自己的需求，输入特定条件来筛选出目标数据。

![](img/1f40781e5feda1ab95e61276e7c80a21_6.png)

## 多样化的筛选条件

![](img/1f40781e5feda1ab95e61276e7c80a21_8.png)

由于每位用户的需求各不相同，软件提供了强大的筛选功能来满足这些个性化需求。

以下是软件支持筛选的部分产品类型示例：
*   FBA跟卖的产品
*   FBM跟卖的产品
*   没有品牌备案的产品
*   跟卖TM标（商标申请中）的产品
*   跟卖R标（已注册商标）的产品
*   当前有库存的产品
*   僵尸链接（长期无销售或卖家消失的链接）
*   有多个卖家的产品链接

![](img/1f40781e5feda1ab95e61276e7c80a21_10.png)

此外，你还可以根据产品的不同排名、价格区间等多种维度进行筛选。

因此，不必再询问软件能否选到某种特定产品。只要你是做跟卖的，无论采用何种运营模式，针对何种产品类型，软件都能通过条件筛选将其采集出来。

![](img/1f40781e5feda1ab95e61276e7c80a21_12.png)
