# 亚马逊店铺安全教程：P1：谨防店铺信息泄露风险 🔒

在本节课中，我们将学习如何检查和清理亚马逊卖家后台的第三方授权，以防止店铺信息被未知或不再需要的应用程序或服务商获取，从而保护你的账户安全。

## 概述

保护亚马逊店铺信息安全至关重要。一个常见的风险点是卖家在后台无意中授权了第三方应用程序或服务商。如果不定期管理这些授权，可能导致店铺信息泄露。本节将指导你如何识别并禁用这些潜在的风险授权。

## 检查第三方授权

上一节我们概述了信息泄露的风险，本节中我们来看看具体的操作步骤。你需要定期检查亚马逊卖家后台的“用户权限”设置。

1.  登录你的亚马逊卖家后台。
2.  进入“设置”菜单，选择“用户权限”。
3.  在“用户权限”页面中，找到“第三方开发人员和应用程序”部分。
4.  点击“管理您的应用程序”按钮。

完成以上步骤后，系统将展示所有已获得你店铺访问权限的第三方应用程序和服务商列表。

## 识别与清理风险授权

在查看了授权列表后，下一步是进行风险评估和清理。如果你曾使用过代运营服务、店铺申诉服务、品牌修改或备案服务等，相关服务商可能会出现在此列表中。

以下是清理未知或不再需要授权的步骤：
*   **仔细审查列表**：确认每一个授权机构的名称和用途。
*   **评估风险**：对于任何你不认识、不记得或不再需要的授权，应视为潜在风险。
*   **禁用授权**：将可疑或不再需要的授权状态调整为“**已禁止**”。

**核心操作代码示意**：
```plaintext
授权状态.更新（“未知应用程序”， “已禁止”）
```

通过此操作，你可以有效切断这些第三方对你店铺数据的访问权限。

## 总结

本节课中我们一起学习了如何防范亚马逊店铺信息泄露。关键点是定期访问后台的“用户权限”设置，检查“第三方开发人员和应用程序”列表，并**禁用所有不明或已不需要的授权**。养成定期检查的习惯，是保障店铺基础安全的重要一环。尚未检查过的卖家，建议立即按照本教程清理授权。
