# 小程序开发基础：P1：定制源代码与注释说明

在本节课中，我们将要学习关于定制化软件交付的核心内容，特别是源代码的交付标准以及代码注释的规范。了解这些内容，有助于你明确在项目交付时能获得什么，以及如何有效地利用这些交付物。

## 源代码交付标准

上一节我们介绍了课程主题，本节中我们来看看定制软件的交付形式。所有定制的系统，都是以源代码的形式进行交付的。交付时，你会获得完整的源代码文件以及一份详细的部署文档，用于指导你将系统部署到服务器或相应的运行环境中。

## 代码注释规范

理解了交付形式后，我们来看看源代码内部的具体情况。关于代码中的注释，需要特别说明：我们的注释只在关键的地方进行添加。

以下是添加注释的几个主要场景：

*   **文件说明**：注释会说明当前这个源代码文件的主要功能和职责。
*   **类与结构说明**：注释会解释这个类或数据结构是做什么用的。
*   **方法/函数说明**：对于重要的方法或函数，注释会描述其作用、参数和返回值。
*   **关键逻辑说明**：对于实现核心业务或复杂算法的代码段，会有相应的注释进行说明。

## 对阅读者的基本假设

然而，需要特别注意一点：我们默认所有能接触到源代码的人，都具备基础的编程能力，能够阅读理解代码的内容。

因此，如果你期望看到一个每行代码、每个步骤都配有详细注释的程序，那么我们的交付标准可能无法满足你的要求。我们提供源代码，更多的是为你提供一份安全的保障和所有权凭证，确保你对项目拥有完全的控制权。

## 给初学者的建议

基于以上规范，这里有一些实用的建议。通常情况下，我们也不建议你在不懂编程的情况下去随意修改这些相关的源代码。

直接修改代码可能导致程序出现错误、功能异常或安全漏洞，造成不必要的问题。如果你有新的功能需求或修改想法，更稳妥的方式是联系我们的开发团队进行沟通和评估。
