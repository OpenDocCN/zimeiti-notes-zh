# 小红书运营教程：P31：第1节：小红书运营之账号定位（上） 📚

在本节课中，我们将要学习小红书平台的流量入口与核心推荐机制，并初步了解账号定位的重要性。掌握这些基础知识，是后续高效运营和引流的前提。

## 流量入口解析 🚪

![](img/5e3147412ac9d79a6fa84531ce981986_1.png)

想要通过小红书获取流量，首先需要搞清楚小红书的流量入口有哪些，以及哪些是需要重点关注的。这有助于我们将精力集中在能快速见效的渠道上。

以下是小红书主要的三个流量渠道：

![](img/5e3147412ac9d79a6fa84531ce981986_3.png)

1.  **首页推荐**：这是小红书最大的流量入口，由关注页、地区页和发现页构成。
    *   **关注页**：用户关注博主后，博主发布的笔记会通过此入口推荐给用户。
    *   **地区页**：系统根据用户定位进行同城内容推荐。
    *   **发现页**：这是所有流量入口中占据绝对地位的入口。对于新账号，发现页基本占据百分之百的流量推荐比重。小红书的推荐机制也主要针对发现页进行。

![](img/5e3147412ac9d79a6fa84531ce981986_5.png)

2.  **搜索推荐**：指用户主动搜索关键词后，系统根据内容匹配度和活跃度进行综合排序得出的结果。与首页推荐（系统主动推荐）不同，搜索是用户主动行为的结果。

![](img/5e3147412ac9d79a6fa84531ce981986_7.png)

![](img/5e3147412ac9d79a6fa84531ce981986_9.png)

3.  **其他来源**：主要指用户通过分享链接直接打开等方式产生的流量。对于新号而言，此渠道难以在短期内带来大量流量。

![](img/5e3147412ac9d79a6fa84531ce981986_11.png)

通过后台数据可以发现，一篇爆款笔记的流量绝大部分（例如95%）来自首页推荐（尤其是发现页），搜索和其他来源占比较小。因此，运营初期应重点关注如何让笔记在发现页获得推荐。

## 核心推荐机制 ⚙️

弄清推荐机制，就能明白一篇笔记是如何获得更大曝光的。小红书的推荐机制主要分为两大块。

![](img/5e3147412ac9d79a6fa84531ce981986_13.png)

### 个性化推荐（用户消费推荐）

![](img/5e3147412ac9d79a6fa84531ce981986_15.png)

所谓个性化推荐，是指系统根据用户在平台上的行为（如点击、关注、评论、收藏、停留时长、搜索），为用户打上对应的“喜好标签”，并据此推荐相关内容。

![](img/5e3147412ac9d79a6fa84531ce981986_17.png)

其核心逻辑包含两个方面：
*   **用户标签**：每个用户可能有多个标签，系统根据其行为动态匹配内容。
*   **笔记标签**：系统会分析我们发布笔记的**标题、封面、文案及内容**，为其打上标签，并尝试推荐给带有对应喜好标签的用户。

**公式表示**：`推荐内容 = f(用户行为历史 -> 用户标签， 笔记内容特征 -> 笔记标签)`

![](img/5e3147412ac9d79a6fa84531ce981986_19.png)

### 推荐机制运转流程

一篇笔记的推荐流程可以简化为以下步骤：

1.  **发布与审核**：笔记发布后，首先进入系统审核阶段。
2.  **初始推荐（冷启动）**：审核通过后，系统会给予一个基础的曝光量（通常为**100到200**的播放量）。前提是账号状态正常且笔记无违规。
3.  **数据评估**：系统根据这波初始流量带来的用户反馈（点击率、点赞、评论、收藏、停留时长等）综合评估笔记质量。
4.  **流量池晋级**：如果数据反馈达到一定标准，笔记会被推荐到更大的流量池，获得更多曝光。反之，推荐则可能停止。

![](img/5e3147412ac9d79a6fa84531ce981986_21.png)

**代码逻辑示意**：
```python
if 笔记审核通过 and 账号状态正常:
    分配初始流量(100~200曝光)
    收集用户交互数据(点赞、评论、收藏等)
    if 综合数据指标 >= 晋级阈值:
        推荐至更大流量池
    else:
        停止推荐
```

值得注意的是，内容质量高的笔记，可能在**20到30分钟**内就快速突破基础播放量，进入下一流量池。

![](img/5e3147412ac9d79a6fa84531ce981986_23.png)
