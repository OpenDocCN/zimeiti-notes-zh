## 概述

在本节课中，我们将要学习如何从零开始获取并积累种子用户。这是启动任何裂变营销活动的基础，没有种子用户，后续的裂变增长就无从谈起。我们将重点探讨如何精准定位你的目标用户，以及如何通过研究平台和设计流程，持续、批量地获取高质量的种子用户。

## 种子用户的重要性与获取平台分类

上一节我们介绍了流量循环体系的整体框架，本节中我们来看看如何获取最关键的启动资源——种子用户。

种子用户是裂变活动的起点。如果没有种子用户，就会出现“巧妇难为无米之炊”的情况。获取种子用户主要依赖平台流量。我们可以将平台分为两大类：

*   **APP类平台**：例如知乎、豆瓣、百度贴吧等独立的应用程序。
*   **微信群平台**：以微信群为载体的各类社群，如宝妈群、运营交流群、创业群等。每一个微信群都可以视为一个独立的流量平台。

## 第一步：精准的流量定位

获取流量的第一步是明确你的目标用户在哪里。这与我们第一天课程中“理解流量”的概念一脉相承。流量定位至关重要。

每个人的业务和产品不同，目标用户群体也不同，因此获取流量的平台也会有巨大差异。为了精准定位，在进入任何一个平台前，建议使用以下两种工具进行分析：

### 1. 使用百度指数分析人群画像

这个工具可以帮助我们了解潜在用户的基本特征，主要关注两点：

*   **年龄分布**：你的目标用户主要处于哪个年龄段？例如，如果你的产品面向30-40岁人群，那么去18-25岁的群体中推广，效果必然不佳。
*   **性别比例**：你的产品用户是男性居多还是女性居多？如果男性占比90%，却去女性社群推广，同样难以启动。

**核心操作**：在百度指数中输入你的行业或产品关键词，查看“人群画像”数据。

### 2. 使用七麦数据（原ASO100）发现更多流量池

这个工具的主要目的是通过一个已知的流量平台（种子平台），找到更多同类型或相关联的平台（竞品平台）。

**核心操作**：
1.  在七麦数据搜索你选定的平台（如“豆瓣”）。
2.  查看该平台公司的所有产品（如豆瓣小组、豆瓣广播等）。
3.  重点研究平台分析报告中列出的“竞品”应用。

以下是筛选有价值竞品平台的方法：

1.  **下载并体验**：将找到的所有竞品APP下载到手机，逐一注册并体验。
2.  **评估付费意愿**：做流量的最终目的是**赚钱**。因此，我们要找的用户不仅要符合年龄和性别特征，还必须具备**付费意愿**。愿意为知识、内容付费的用户，通常付费意愿更强。
3.  **深度研究平台**：访问“人人都是产品经理”网站或APP，搜索你感兴趣的竞品平台名称（如“豆瓣”）。你会看到大量产品经理撰写的深度分析文章。

通过阅读这些文章，你可以理解：
*   该平台的核心用户是谁？
*   平台的发展方向和规则逻辑是什么？
*   该平台是否适合进行知识付费类产品的推广？

结合以上分析，你可以从众多竞品中筛选出2-3个最符合你目标用户画像和付费意愿的平台，作为你重点攻克的对象。

**总结**：完成以上用户与平台研究后，你就明确了目标。剩下的工作就是坚定地执行。

## 第二步：微信群——高效的种子用户流量池

微信群是获取种子用户非常高效的渠道。我们可以设计一个标准化的操作路径，实现种子用户的批量获取和循环增长。

这个标准化路径包含以下五个步骤：

1.  **流量定位**：明确你要在群里吸引什么样的用户。
2.  **找群**：找到包含目标用户的微信群。
3.  **设置诱饵**：设计一个吸引人的“诱饵”，引导群成员添加你的微信。（下节课会详细讲解诱饵设计）
4.  **计算转化**：统计通过诱饵添加你的人数，计算转化率。
5.  **主动添加与循环**：对于未添加你的群成员，主动添加他们。添加后，立即将其拉入你建立的“课程群”或“福利群”，并在新群中再次设置诱饵，形成循环。

这个路径的核心在于 **“找群”** 。只要有源源不断的优质微信群，这个循环就能持续运转。

### 如何高效“找群”？

我把找群的方法分为两类：

**第一类：搜索群（推荐）**

*   **内部搜索**：利用微信自身的搜索功能。在微信顶部的搜索栏，输入你定位的关键词（如“运营”、“新媒体”、“创业”），选择“朋友圈”范围。你会看到好友分享的群二维码或引流入群的信息。每天坚持搜索，能稳定获得少量但精准的群资源。
*   **外部搜索**：在目标用户聚集的APP内搜索。例如，在豆瓣小组、知乎话题、千聊、荔枝微课等平台，直接搜索“XX交流群”、“加入社群”等关键词，可以找到大量公开的群组信息或引流帖子。

**第二类：截流群（了解即可，不推荐操作）**

这种方法指收集别人正在做裂变活动的海报，用小号扫码进入别人的裂变群，然后在群内冒充官方人员，推广自己的联系方式进行引流。这种方法风险高，易引起纠纷，不建议使用。

## 第三步：设计种子用户的自动循环增长体系

仅仅获取用户还不够，我们需要设计一个能自动运转、循环带来新种子用户的体系。

关键玩法设计如下：

1.  **即加即拉群**：无论用户是主动添加你，还是你主动添加用户，一旦通过好友验证，**立刻将其拉入一个预设的群**。
2.  **群内自动化授课**：在群内，使用“微兔”等工具设置定时自动发送课程内容（可以是录播的语音或文字），实现无人值守的“讲课”。
3.  **设置课程“埋点”**：在课程结束时，设计一个“解锁下一节课”的任务，通常要求用户**转发指定海报和话术到朋友圈**。
4.  **闭环循环**：用户转发后，其朋友圈带来的新用户会添加你。添加后，重复第1步：**立刻拉入新群**，然后开始新一轮的自动化授课。如此循环往复，种子用户就能自动带来新的种子用户。

这个体系的核心是**将流程产品化、自动化**，使其可以被批量复制，减少对人力的依赖，实现流量的持续循环增长。

## 克服执行障碍与实现批量化

很多人知道方法却无法坚持，主要原因有四点：目标不明确、反馈弱（加人少）、转化低、惰性强。其中，**惰性（执行力差）** 是最核心的障碍。

如何克服？关键在于**设计即时盈利点**，建立以赚钱为目标的正向反馈循环。例如：
*   设计**1元低价产品**，快速获得成交的成就感。
*   设计**转发任务**，看到新增好友的即时反馈。
*   设计**保证金产品**，即使最终退还，资金在账期间的掌控感也能带来激励。

当执行动作能带来直接的经济或流量反馈时，坚持就不再困难。

若要实现**批量化**获取种子用户，需要配置以下资源：
1.  **批量微信号**：用于承载流量。
2.  **批量微信群**：通过教授方法，让团队成员或合作伙伴帮你持续找群。
3.  **批量执行人员**：与执行者建立“利益共同体”，绑定收益关系。
4.  **批量手机设备**：用于管理多个微信号。
5.  **账号实名解决方案**：部分微信功能需要实名认证，这是批量操作中的一个技术环节。（注：相关敏感资源需通过特定渠道合规解决）
