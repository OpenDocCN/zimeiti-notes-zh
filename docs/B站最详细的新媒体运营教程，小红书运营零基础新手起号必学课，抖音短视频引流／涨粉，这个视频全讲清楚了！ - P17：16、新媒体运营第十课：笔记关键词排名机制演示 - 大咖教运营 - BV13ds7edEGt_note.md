## 第一步：内容审核

笔记发布后，系统首先会进行内容审核。审核主要关注两个方面：

以下是审核的核心检查点：
*   **违规内容**：检查笔记中是否包含敏感词、平台限制准入的类目或其他非法信息。
*   **原创性与质量**：检查笔记是否存在抄袭、素材高度重复或非原创的情况。

当你的内容审核过关后，流程进入下一步。

## 第二步：笔记收录

审核通过后，笔记进入收录环节。笔记能否被收录，取决于两个核心因素：

以下是决定收录的关键因素：
*   **账号权重**：账号在系统流量层级中的位置，权重越高，通常基础曝光越好。
*   **笔记质量**：由用户的行为数据决定。即使账号权重高，如果笔记的点击率等互动数据差，也可能不被收录。

**公式示例**：`笔记收录概率 = f(账号权重, 用户互动数据)`

当笔记被收录的同时，系统会开始对其内容进行分析。

## 第三步：特征提取与打标签

系统会分析笔记内容，提取关键特征并为笔记打上标签。这决定了笔记会被归类到哪些搜索关键词下。

标签来源主要包括以下几个方面：

以下是系统提取标签的主要依据：
*   **用户标签**：阅读笔记的用户画像和兴趣标签。
*   **账号历史数据**：账号以往内容被打上的标签。
*   **内容关键词**：从标题、正文中自动拆分和提炼出的核心词汇。

**代码逻辑示意**：
```python
# 伪代码：系统提取笔记标签
def extract_note_tags(note_content, user_feedback, account_history):
    tags_from_keywords = analyze_keywords(note_content) # 分析内容关键词
    tags_from_users = analyze_user_behavior(user_feedback) # 分析用户行为反馈
    tags_from_history = get_account_tags(account_history) # 获取账号历史标签
    final_tags = combine_tags(tags_from_keywords, tags_from_users, tags_from_history) # 综合得出最终标签
    return final_tags
```

标签提取完成后，系统会进入核心的索引建立阶段。

## 第四步：内容索引与动态排名

系统会根据提取的标签，在内部的“索引库”中为笔记在各个关键词下预先进行排序。这不是实时计算，而是提前完成的。

为了更直观地理解，我们模拟一篇笔记的索引过程。

### 排名的动态调整

![](img/4e1024b8f17a79b96f7129433e2e04ec_23.png)

![](img/4e1024b8f17a79b96f7129433e2e04ec_25.png)

初始排名并非一成不变。系统会根据笔记在不同标签下的**实际数据表现**（即上一节讲的点击率、搜索跳出率等）进行动态调整。

![](img/4e1024b8f17a79b96f7129433e2e04ec_27.png)

排名调整遵循以下规律：

以下是排名动态调整的规则：
*   **数据好则排名上升**：如果在“云南旅游”下点击率高，排名可能从D位上升到B位甚至A位。
*   **数据差则排名下降或移除**：如果在“旅游”这个大词下表现一般，排名可能逐渐下降。在“学生”这类弱相关标签下，若长期无数据支撑，笔记可能会被**完全移出**该关键词的排名库。

### 动态排名机制

新媒体平台比传统搜索引擎更智能。为了平衡流量、避免内容同质化，平台会采用 **“动态排名”** 机制。

这意味着，对于同一个搜索词，系统可能准备了**多个不同的索引库版本**，根据搜索用户的个人偏好进行微调展示。因此，不同用户搜索“云南旅游”时，看到的笔记顺序可能略有不同。
