## 流量入口解析 🚪

![](img/f40100798d5856da86e3d2c6d56528f5_1.png)

想要通过小红书获取流量，首先需要搞清楚小红书的流量入口有哪些，以及哪些是需要重点关注的。这有助于我们将精力集中在能快速见效的流量入口上。

以下是小红书主要的三个流量渠道：

![](img/f40100798d5856da86e3d2c6d56528f5_3.png)

1.  **首页推荐**：这是小红书最大的流量入口，由关注页、地区页和发现页构成。
    *   **关注页**：用户关注博主后，博主发布的笔记会通过此入口推荐给用户。
    *   **地区页**：系统根据用户定位进行同城内容推荐。
    *   **发现页**：这是所有流量入口中占据绝对地位的入口。对于新号而言，发现页基本占据百分之百的流量推荐比重。小红书的推荐机制也主要针对发现页进行。

2.  **搜索推荐**：指用户主动搜索关键词后，系统根据内容匹配度和活跃度进行综合排序得出的结果。与首页推荐（系统主动推荐）不同，搜索推荐是用户主动行为的结果。

![](img/f40100798d5856da86e3d2c6d56528f5_5.png)

![](img/f40100798d5856da86e3d2c6d56528f5_7.png)

3.  **其他来源**：主要指用户通过分享链接直接打开笔记等方式。对于新号而言，此渠道难以在短时间内带来大量流量。

![](img/f40100798d5856da86e3d2c6d56528f5_9.png)

![](img/f40100798d5856da86e3d2c6d56528f5_11.png)

通过后台数据图可以清晰看到，一篇爆款笔记的流量绝大部分（例如95%）来自首页推荐，尤其是发现页。因此，运营初期应重点关注如何让笔记在发现页获得推荐。

## 核心推荐机制 ⚙️

上一节我们介绍了流量入口，本节中我们来看看小红书的两大推荐机制。弄清推荐机制，就能明白一篇笔记是如何获得更大曝光的。

这两大推荐机制分别是：

![](img/f40100798d5856da86e3d2c6d56528f5_13.png)

*   **个性化推荐（用户消费推荐）**
*   **关键词搜索推荐**

![](img/f40100798d5856da86e3d2c6d56528f5_15.png)

### 个性化推荐机制

![](img/f40100798d5856da86e3d2c6d56528f5_17.png)

个性化推荐，也称为用户消费推荐，其核心逻辑如下：

![](img/f40100798d5856da86e3d2c6d56528f5_19.png)

1.  **用户行为产生标签**：用户在浏览小红书时产生的点击、关注、评论、收藏、停留时长、搜索等行为，会被系统记录并用于为用户打上对应的“喜好标签”。
2.  **系统匹配推荐**：系统根据用户的喜好标签，为其推荐相对应的内容。每个用户可能有多个标签，系统会动态匹配。
3.  **笔记标签匹配**：系统会根据发布笔记的标题、封面、文案及内容，为笔记打上“笔记标签”。
4.  **双向匹配与评估**：系统将带有特定标签的笔记，推荐给带有对应喜好标签的用户。再根据这批用户的反馈（点击、点赞等）来评估笔记质量，决定是否将其推荐给更多用户。

**公式表示**：`推荐结果 = 用户喜好标签 ∩ 笔记内容标签`

### 笔记推荐流程

以下是笔记发布后的核心推荐运转流程：

![](img/f40100798d5856da86e3d2c6d56528f5_21.png)

1.  **发布与审核**：笔记发布后，首先进入系统审核阶段。
2.  **基础流量测试**：审核通过后，系统会给予笔记**100到200**的基础播放量。前提是账号状态正常且笔记无违规。
3.  **数据反馈评估**：系统根据这波基础流量中用户的反馈（点击率、点赞、评论、收藏、停留时长等）综合评估笔记质量。
4.  **流量池晋级**：如果笔记数据反馈达到标准，就会进入下一个更大的流量池，获得更多曝光。如果内容质量足够好，可能在20-30分钟内就能突破基础流量池。

**代码逻辑描述**：
```python
if 账号状态正常 and 笔记无违规:
    笔记进入审核流程
    if 审核通过:
        分配初始流量池(100-200曝光)
        监测用户互动数据(点击率、点赞、评论等)
        if 数据反馈 >= 系统阈值:
            晋级至更大流量池
            重复监测与晋级流程
        else:
            推荐量逐渐减少
```
