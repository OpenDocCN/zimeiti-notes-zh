## 单店模式商品规划 🏪

上一节我们介绍了商品管理的两种模式，本节中我们来看看**单店模式**的具体规划。单店模式需要更精细化的产品管理。

### 店铺商品基础数量需求

首先，我们需要了解店铺整体对商品基础数量的需求。对于整个小红书店铺而言，商品数量最好保持在 **7到15个** 之间。

这个数量范围基于各大电商平台的通用数据逻辑。在单店模式下，商品结构通常包含：
*   **一个主推款**
*   **几个副推款**
*   **其他不主推的补量产品**

保持这个数量范围的目的在于：
1.  让店铺看起来内容充实，避免主产品显得孤单。
2.  提升店铺的**属性权重**。店铺权重由动销率、营销率、成交金额等多维度数据构成。**必须先有店铺权重，才能支撑单品权重**。因此，我们需要先规划好整体店铺的商品布局。

![](img/5dae9cda8134a65c8f957c4d44eed226_5.png)

![](img/5dae9cda8134a65c8f957c4d44eed226_7.png)

**核心公式：店铺商品总数 = 1个主产品 + 2-4个副产品 + 5-10个补量产品 ≤ 15个**

实际操作中，最低数量不应少于7个，最大数量不建议超过15个。后期可以打造多个爆款，但总商品数仍建议控制在此范围内。当成功打造数个爆款后，可考虑开设新店进行复制，而非无限扩充单一店铺的商品。

### 商品发布技巧

商品发布并非随意上架，而是有特定技巧的。以下是具体的操作步骤：

首先，了解发布节奏。建议先上传**1到2件副产品**，且每天上架新产品不超过3件。

让店铺带着这些副产品正常运行一天后，再上架主产品。这样做的原因是：小红书系统需要时间录入新产品并给予基础排名。等待一天后，系统已完成初始排名，此时再操作主产品数据，能更有效地提升整体排名。**第一天只需上架副产品，无需进行任何数据操作。**

其次，掌握上架时间。时间选择对商品曝光至关重要。

*   **副产品**：应**避开行业流量高峰期**。用竞争力不强的副产品在高峰期与别人的主产品竞争是无效的。应选择冷门时间段获取额外流量，但需避开完全无流量的时段（如凌晨2点至5点）。
    *   **示例**：以女装为例，高峰期通常为上午10点至下午1点、下午5点至7点、晚上8点至11点。上架副产品应避开这三个高峰段。
*   **主产品**：应在**高峰期开始前30分钟**上架。
    *   **原因**：① 避开与同行产品的正面交锋；② 给予系统约30分钟的反应和数据处理时间。前期应利用这30分钟获取高峰期的前置流量，待店铺数据稳定后，后续的主产品方可尝试在高峰正点上架竞争。

**代码示例（理想上架逻辑）：**
```python
if 产品类型 == “副产品”:
    上架时间 = 选择(非高峰时段)
elif 产品类型 == “主产品” and 是否为店铺第一个主产品:
    上架时间 = 行业高峰期 - 30分钟
else:
    上架时间 = 行业高峰期 # 后续主产品可尝试竞争高峰流量
```

## 店群模式商品规划 🔄

![](img/5dae9cda8134a65c8f957c4d44eed226_9.png)

![](img/5dae9cda8134a65c8f957c4d44eed226_11.png)

了解完相对复杂的单店模式后，本节我们来看看操作更简单但需要耐心的**店群模式**。

![](img/5dae9cda8134a65c8f957c4d44eed226_13.png)

店群模式与单店模式的主要区别在于：它不需要复杂的商品规划，核心是**持续大量上架商品**。

以下是店群模式的执行要点：
*   每个店铺每天上架约 **50个** 产品。
*   在上架大量商品前，必须确保店铺有 **破零** 订单（即完成首笔交易）。店铺未破零，平台几乎不会分配流量。
*   破零操作无需针对所有商品，只需选择其中2-3个产品完成即可。
*   之后便可按计划每日上架商品，平台会正常分配流量。
*   此模式需要在多个店铺（例如10个）中重复此操作。可利用小红书后台的“一键代发”、“一键上传”等功能提升效率，但对执行者的耐心要求较高。

![](img/5dae9cda8134a65c8f957c4d44eed226_15.png)

本节课中我们一起学习了小红书商品上架的两种核心模式：追求精细化运营的**单店模式**和以量取胜的**店群模式**。关键点包括单店商品数量控制在7-15个、掌握副产品与主产品的上架节奏与时间技巧，以及店群模式每日上架约50款商品并确保破零的流程。理解这些内容后，进行商品上架操作将会更加得心应手。
