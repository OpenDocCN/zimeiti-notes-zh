在本节课中，我们将深入探讨小红书平台如何为用户打上标签，以及平台主要的流量分发入口。理解这些机制，是运营好小红书账号、获取精准流量的关键。

上一节我们介绍了系统如何判定内容标签，本节中我们来看看平台如何判定用户标签。这个过程能帮助我们分析自己的笔记是如何被系统推荐给用户的，并反推背后的逻辑，从而让我们的笔记被更多精准用户看到。

## 用户标签判定机制

系统为用户打标签的主要依据是用户的**浏览行为**。用户注册时选择的兴趣范围参考价值不大，它仅作为初期内容方向的引导，无法进入系统深层的用户标签池。系统判定用户兴趣分类的核心依据是以下行为数据：

以下是判定用户标签的几个关键行为指标：

1.  **点击率**
    *   **公式**：`点击率 = (点击某类内容的次数) / (系统推送的该类内容总次数)`
    *   系统在发现页向用户推送多种内容（如美妆、家居、母婴）。用户点击哪类内容更多，系统就能计算出其偏好。例如，用户点击10条内容，其中美妆3次、家居2次、母婴0次，系统会判定该用户对母婴内容不感兴趣，并为其打上“美妆”的强关联标签。

2.  **停留时长**
    *   用户点开笔记后的观看时长。如果用户对某类内容（如美妆）普遍快速划走，说明推送的具体方向（如“上粉”）可能不匹配。反之，如果用户对偶然推送的某篇笔记（如家居装修）停留很久、仔细阅读，系统会判断用户可能有新的潜在需求（如“装修”或“购买盘子”），从而调整标签。

3.  **页面浏览量（PV）**
    *   用户浏览页面的深度。例如，用户不仅看了某篇笔记，还点进博主主页，并连续看了第二、第三篇笔记。提升用户在你主页的浏览深度，能显著增加账号的健康度评分，对账号成长有巨大帮助。

4.  **互动行为**
    *   包括点赞、收藏、评论、关注。这些数据直接决定了笔记能否进入下一层更大的流量推荐池。互动数据高的笔记，获得推荐的机会更大，也能带动账号内其他笔记的流量，是运营中需要持续优化的核心目标。

通过以上行为，系统会为用户构建一个动态的标签云图。例如，一个用户的“美妆”标签最大，代表他是美妆内容的深度爱好者，系统便会持续为其推荐相关内容，以提升用户的获得感与平台停留时长。

## 平台流量分发入口

理解了用户标签，我们再来看看小红书主要的流量分发入口。作为用户你可能经常使用，但作为运营者需要分析其区别。

小红书平台主要有四个流量入口：

以下是四个主要流量入口的详细说明：

1.  **发现页**
    *   这是笔记获取流量的**主要来源**。其内容包含：
        *   已关注达人发布的新内容。
        *   系统根据用户标签（点击率、停留时长等）推荐的、判定用户可能喜欢的“兴趣匹配内容”。
        *   系统为了平台内容多样化而推荐的“发散性内容”，即用户可能未接触过但潜在感兴趣的内容。

2.  **关注页**
    *   这里集中展示所有已关注博主发布的笔记。用户通常从这里直接查看核心关注者的动态。

3.  **附近页**
    *   根据手机定位，推送同城或附近的内容。此入口以具有强地域特征的内容为主，如**美食、本地旅游、线下店推广**等。如果你是线下实体店（如美容院、餐厅）的运营者，需要特别重视这个入口，并在后期考虑信息流广告时，注意针对附近人群进行推送。

4.  **购物页**
    *   这是小红书向电商化发展、实现商业变现的核心入口。主要满足用户的直接购物需求。目前，用户习惯仍在培养中，许多人仍倾向于通过搜索笔记来“种草”，而非直接进入商城购物。
