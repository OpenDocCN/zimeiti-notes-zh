### 内容审核与标签匹配

上一节我们介绍了账号注册与定位，本节中我们来看看内容发布后，系统是如何处理的。

用户和内容都会被系统打上相应的标签。例如，在发送笔记时，系统会自动为笔记打上标签。每篇笔记都会先经过机器审核，检测内容是否违规。如果机器审核未发现违规，笔记就会进入大数据算法匹配分发流程。

如果系统检测到疑似违规内容，但算法程序无法确定，就会将内容提交给人工审核。人工审核确认无误后，笔记才能进入分发匹配流程。如果笔记被机器和人工均判定为违规，账号可能会受到限制发言等处罚。

此外，在推荐时，系统会提取笔记中的关键词、图片标签、话题等信息，将笔记分类到对应的类目中。用户在一开始注册小红书时，平台会要求选择地理位置和兴趣领域，这正是为了给用户的兴趣打上标签，以便后续根据标签匹配和分发相关内容。

### 核心推荐机制：流量池模型

很多人觉得小红书的推荐机制很奇怪，比如一篇笔记的阅读量会在不同时间段持续增长。这其实源于其核心的流量池推荐机制。

为了便于理解，我们可以通过以下模型来说明：

1.  **初始分发**：笔记发布后，系统会将其分配到一个**基础流量池**。这个池的流量通常不大，可能只有一两百或两三百个用户会被推荐看到这篇笔记。
2.  **数据评估**：系统会根据基础流量池内用户的**互动参数**（主要是点赞、收藏、转发）来评估笔记质量。
3.  **流量升级**：如果笔记的互动参数达到一定**阈值**，系统就会将其推荐到下一个更大的**流量池**（例如1000人）。
4.  **逐级放大**：在更大的流量池中，如果互动参数依然表现良好，达到新的阈值，笔记会被推荐到再下一个流量池（例如5000人）。如此循环，流量池规模可能逐步扩大到1.5万甚至更多。
5.  **推荐终止**：如果笔记在某一级流量池中的互动参数未能达到进入下一级的阈值，推荐就会停止在该层级。

**核心公式**：
```
笔记进入下一级流量池 = 当前流量池互动数据 > 系统设定阈值
```

以下是流量池模型的演进过程示意：
*   **第一级**：基础流量池（~200-300曝光）
*   **第二级**：达标后进入更大流量池（~1000曝光）
*   **第三级**：继续达标后进入更大流量池（~5000曝光）
*   **第四级及以后**：持续达标，流量池逐级放大

小红书的推荐不是一蹴而就的，而是笔记通过**逐步积累良好的用户互动数据**，从而**一级一级获得更大曝光**的过程。不存在笔记一发出去就立刻获得海量曝光的情况。
