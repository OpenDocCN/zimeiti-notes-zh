## 第一步：内容审核 🔍

笔记发布后，系统首先会进行内容审核。审核主要关注以下两点：

以下是审核的核心检查项：
1.  **违规内容**：检查笔记中是否包含敏感词、平台限制准入的类目或其他非法信息。
2.  **原创性与质量**：检查笔记是否存在抄袭、素材高度重复或非原创的情况。

![](img/c10997f62038e2c0da93283996ca2e19_11.png)

当内容审核通过后，笔记才会进入下一个环节。

---

## 第二步：笔记收录 ✅

笔记能否被系统收录，取决于两个核心因素：**账号权重**和**笔记质量**。

*   **账号权重**：由账号在平台流量层级中的位置决定，权重有高有低。
*   **笔记质量**：由**用户的行为数据**决定。例如，笔记的点击率（小眼睛数量）是重要指标。

![](img/c10997f62038e2c0da93283996ca2e19_13.png)

**公式**：`笔记质量 ∝ 用户互动行为（点击、点赞、收藏等）`

![](img/c10997f62038e2c0da93283996ca2e19_15.png)

![](img/c10997f62038e2c0da93283996ca2e19_17.png)

即使账号权重高，如果笔记发布后获得的初始曝光中，点击率远低于平均水平（例如1万曝光只有几百点击），系统也会判定该笔记质量不佳，可能不予收录。反之，即使初始曝光量低，但点击率表现良好，被收录的可能性也会增加。

---

## 第三步：特征提取与打标签 🏷️

笔记被收录的同时，系统会对其内容进行分析，提取特征并打上标签。

系统会从多个维度提取笔记特征，从而判断其内容类别：

以下是标签提取的主要数据来源：
1.  **用户标签**：浏览、互动过该笔记的用户身上带有的兴趣标签。
2.  **账号历史数据**：该账号以往发布内容被系统打上的标签。
3.  **内容关键词**：从笔记标题、正文中拆分和提炼出的核心词汇。

例如，一篇旅游笔记可能被喜欢“旅游”、“云南”、“攻略”的用户点击，系统便会从这些用户行为和内容关键词中，提取出“云南旅游”等标签，并给笔记打上。

---

## 第四步：内容索引与初始排名 📈

系统会根据上一步提取的标签，在内部的“索引库”中，为笔记在各个相关关键词下预先建立一个排名。**这不是用户搜索时才临时排序，而是收录后即完成的预处理。**

![](img/c10997f62038e2c0da93283996ca2e19_19.png)

为了更直观地理解，我们模拟一篇标题为《8月去云南旅游的完整攻略，学生党必看，完整避坑攻略》的笔记。

![](img/c10997f62038e2c0da93283996ca2e19_21.png)

系统可能从中提取出“云南”、“旅游”、“云南旅游”、“学生”、“避坑”等标签。假设在系统的索引库中，这篇笔记（记为G笔记）在不同标签下的初始排名如下：

以下是笔记G在不同标签下的模拟初始排名：
*   **在“云南”标签下**：排名第7位（在A、B、C、D、E、F笔记之后）。
*   **在“旅游”标签下**：排名第6位（在F笔记的位置）。
*   **在“云南旅游”标签下**：排名第4位（在D笔记的位置）。
*   **在“学生”标签下**：排名第8位。
*   **在“避坑”标签下**：排名第1位。

这个初始排名不是一成不变的，它会根据后续的用户行为数据**动态调整**。

## 第五步：动态排名调整 🔄

索引库中的排名会根据笔记在对应关键词下的实时数据表现（如搜索点击率、用户互动等）进行升降。

**动态调整规则如下：**
1.  **排名上升**：如果笔记在“云南旅游”下的点击和互动数据很好，其在该标签下的排名可能会从第4位上升到第2位。
2.  **排名下降或维持**：在“旅游”这种宽泛标签下，由于内容相关性较弱，数据表现可能一般，排名会缓慢下降，但不会完全消失。
3.  **标签移除**：在“学生”、“避坑”这类弱相关标签下，如果长期没有有效点击和数据支撑，笔记最终会从这些标签的排名中被移除。

此外，平台为了平衡流量、避免内容同质化，并更精准地匹配用户兴趣，采用了 **“动态排名”** 机制。

**代码概念**：
```python
# 简化示例：同一关键词下，不同用户可能看到不同的排序结果
def get_search_results(keyword, user_profile):
    # 根据关键词获取一个基础笔记列表
    base_list = get_notes_from_index(keyword)
    # 结合用户个人画像（兴趣、历史行为）对列表进行个性化重排
    personalized_list = rerank_by_user_preference(base_list, user_profile)
    return personalized_list
```

这意味着，对于“云南旅游”这个热词，不同用户搜索时，看到的笔记排序可能不同。系统会准备多个索引版本，根据实时数据和用户偏好进行微调，让排名不再固定，从而给更多优质笔记曝光机会。

本节课中我们一起学习了小红书笔记关键词排名的完整机制。我们从**内容审核**开始，了解了笔记**收录**的条件，明白了系统如何通过**特征提取**为笔记打上标签，并在此基础上建立**内容索引**和初始排名。最后，我们揭示了排名并非固定不变，而是会根据用户行为数据进行**动态调整**，并且平台会运用**动态排名**策略使结果更智能、更公平。理解这套流程，将帮助你更有针对性地优化笔记，提升其在搜索中的可见度。
