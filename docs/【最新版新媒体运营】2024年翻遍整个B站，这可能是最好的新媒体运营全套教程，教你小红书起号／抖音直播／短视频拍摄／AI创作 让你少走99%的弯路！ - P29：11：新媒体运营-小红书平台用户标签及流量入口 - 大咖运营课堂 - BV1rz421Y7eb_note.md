## 用户标签判定机制 🔍

小红书系统为用户打标签，主要依据是用户的**浏览行为**。用户注册时选择的兴趣范围，参考价值不大，仅作为初期内容方向的引导，无法进入系统深层的用户标签池。

系统根据以下用户行为数据来判定其兴趣分类：

以下是判定用户标签的核心行为指标：

1.  **点击率**
    *   **定义**：系统在发现页向用户推送多种内容（如美妆、家居、母婴等），用户点击某一类内容的频率。
    *   **作用**：系统通过计算用户点击不同类别内容的概率，初步判断其兴趣。例如，用户点击10条内容，其中美妆类3次，家居类2次，母婴类0次，系统会判定该用户对母婴内容不感兴趣，并可能为其打上“美妆”的强关联标签。

2.  **停留时长**
    *   **定义**：用户点开一篇笔记后观看的持续时间。
    *   **作用**：更精细地判断用户喜好。即使同属“美妆”大类，如果用户对某篇护肤笔记停留时间很短，但对一篇关于“盘子”的家居笔记停留很久，系统可能会推断用户近期有购买家居用品（如盘子）的需求，从而调整推荐策略。

3.  **页面浏览量（PV）**
    *   **定义**：用户浏览页面的深度，例如不仅看了单篇笔记，还进入了博主主页并浏览了多篇内容。
    *   **作用**：高PV值表明用户对账号内容有浓厚兴趣。这不仅能提升单篇笔记的权重，还能为整个账号的健康度和评分带来积极影响。公式上可以理解为：`账号健康度 ∝ 用户平均PV值`。

4.  **互动行为**
    *   **定义**：包括点赞、收藏、评论、关注等用户主动操作。
    *   **作用**：这是最直接的正面反馈。高互动数据能显著提升笔记进入更大流量池的几率，带动账号其他笔记的曝光，并积累粉丝，是所有运营者努力提升的核心目标。用代码逻辑可表示为：
        ```python
        if 笔记.互动率 > 阈值:
            系统.提升推荐权重(笔记)
            账号.总权重 += 增益值
        ```

通过以上行为的综合分析，系统会为用户构建一个动态的“兴趣云图”，其中标签的大小和关联强度代表了用户的不同偏好，并据此进行个性化内容推荐。

## 平台流量分发入口 🚪

了解用户标签后，我们还需清楚笔记会被分发给用户的哪些入口。小红书主要有四个流量分发入口。

以下是四个核心流量入口及其特点：

1.  **发现页**
    *   **内容构成**：
        *   已关注达人发布的新内容。
        *   根据用户标签（点击率、停留时长等）匹配的“猜你喜欢”内容。
        *   系统为丰富内容多样性而推荐的发散性内容。
    *   **重要性**：这是笔记获取推荐流量的**主要来源**，也是运营中需要重点攻克和优化的阵地。

2.  **关注页**
    *   **内容构成**：纯粹展示用户所关注的所有博主发布的最新笔记。
    *   **特点**：流量精准但相对稳定，依赖于已有的粉丝基础。

3.  **附近页**
    *   **内容构成**：基于手机定位，推送同城或附近地区的笔记。
    *   **适用内容**：具有强地域特征的内容，如本地美食、旅游攻略、线下店铺（美容院、咖啡馆）推广等。
    *   **运营提示**：如果你是线下实体店，在做信息流广告或内容运营时，应特别注意对此入口的优化和投放。

4.  **购物页**
    *   **内容构成**：小红书电商生态的核心入口，直接导向商品购买。
    *   **现状**：部分用户尚未形成在此直接购物的强习惯，更多仍通过搜索笔记“种草”。但这是平台商业化和博主变现的重要方向。
