# 亚马逊ERP系统教程：P1：无限子账号分身功能详解 🚀

在本节课中，我们将要学习一款专为中小型亚马逊工作室设计的ERP系统。该系统核心优势在于**一次性付费、终身使用**，并支持创建**无限数量的子账号**，有效解决团队协作中的成本与数据整合难题。

## 概述：系统核心价值 💎

![](img/ea6de7596731e57181ca900b4125c276_1.png)

对于3到5人的亚马逊工作室，若为每位成员单独购买ERP系统，将面临高昂成本、数据分散及信息整合困难等问题。本节介绍的ERP系统只需支付一次费用即可终身使用，且无任何隐藏收费。它支持创建大量子账号，可供百人团队同时协作。

## 系统主要功能一览 🔧

上一节我们概述了系统的价值，本节中我们来看看它具体包含哪些功能。该系统功能全面，旨在覆盖亚马逊运营的多个环节。

![](img/ea6de7596731e57181ca900b4125c276_3.png)

以下是该系统提供的主要功能模块：
*   **采集模式**：支持铺货及跟卖两种商品采集方式。
*   **订单管理**：处理订单、安排发货。
*   **供应链管理**：实现采购同步与运费试算。
*   **财务管理**：提供财务数据汇总功能。
*   **辅助工具**：包含多种运营补助工具。

## 核心功能：子账号与权限管理 👥

![](img/ea6de7596731e57181ca900b4125c276_4.png)

了解了基础功能后，我们深入探讨其核心特色——灵活的子账号与权限管理系统。在会员功能的“子账号角色”设置中，主账号可以为子账号分配具体操作权限。

![](img/ea6de7596731e57181ca900b4125c276_5.png)

以下是权限分配的关键点：
*   **权限库丰富**：系统提供上百种功能权限可供自由选择与组合。
*   **按需分配**：可根据子账号成员负责的不同工作内容（如选品、客服、财务），分配差异化的功能权限。
*   **数据隔离与安全**：子账号之间的数据相互独立、互不可见，有效保障了业务隐私。但主账号有权查看所有子账号的全部数据，无需额外进行数据整合。
*   **权限绑定角色**：权限分配通过“角色”来实现。先创建角色并赋予权限，再将角色分配给子账号。

## 如何创建子账号分身 🛠️

我们已经知道如何配置权限角色，接下来看看如何具体创建一个子账号。整个过程在“账号管理”中完成，步骤清晰简单。

创建子账号需按需求填写以下信息：
1.  **基础信息**：填写账号名称、密码等。
2.  **绑定角色**：在“角色”栏选择预先创建好的角色。此账号将仅拥有该角色被分配的功能权限。
3.  **设置授权时间**：授权时长可自由设定，例如3天、一个星期或半年等，灵活性极高。

填写完所有信息后，点击“确定”即可成功创建子账号分身。

## 无限子账号的应用场景 💡

由于该ERP系统的子账号创建**无数量限制**，这为其应用带来了极大的灵活性。

以下是一些典型的应用场景：
*   **内部团队扩展**：完全满足自身工作室团队规模的增长需求。
*   **共享使用**：可以安全地提供给合作伙伴或亲朋好友使用特定功能。
*   **商业变现**：甚至可以将子账号作为一种服务对外出售。
