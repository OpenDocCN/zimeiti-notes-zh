## 商品准备 📦

店铺运营的第一步是准备好商品。商品不仅是销售的主体，也是店铺权重的重要组成部分。

以下是商品准备的具体要求：

*   **商品数量范围**：整体店铺的商品数量应在 **7~45个** 之间。即使是做单品单店，也需要保证在这个数量区间内。
*   **商品构成**：
    *   **主产品**：1件。这是我们主要推广的核心爆款产品。
    *   **副产品**：2~4件。这些是作为第二、第三爆款的备选产品。
    *   **补量产品**：5~10件。这些产品的主要作用不是直接销售，而是为了**增加店铺的商品丰富度、提升店铺活跃度，并辅助提升账号的基础权重**。

商品发布并非一次性完成，而是有技巧地分批上架。

以下是商品发布的技巧：

*   **发布顺序**：先上架1-2件副产品，**主产品暂时不上架**。主产品通常在第二天、第三天或第四天发布。
*   **目的**：先上副产品是为了让小红书系统“认识”并录入我们的店铺，提升账号初始权重，测试账号的流量获取能力。
*   **每日上架量**：每天上架产品不超过3件，通常以1-2件为宜。
*   **上架时间**：
    *   **副产品**：避开行业的流量高峰期上架，避免与主产品竞争初始流量。
    *   **主产品**：选择在行业流量**高峰期前30分钟**上架。这能给系统留出约5-10分钟的录入时间，使产品在高峰期来临时能获得更好的初始排名，从而得到更多曝光。

**公式：** 店铺商品总数 = 1（主产品） + 2~4（副产品） + 5~10（补量产品） = 7~15（初始建议数量）

**核心提示**：店铺商品数量的下限是7件。低于7件，店铺的整体权重很难提升。前期至少保证7件商品在售，后续可以逐步补充至15件或更多。

## 基础数据准备 📊

商品上架后，需要为它们营造一个“受欢迎”的初始状态，这需要通过基础数据来体现。

基础数据主要包括评价、收藏、加购和销量。

以下是基础数据的具体要求：

![](img/79e86260c9b88217ada0a641fd8af643_5.png)

*   **商品评价**：每个商品前期至少需要**5个**评价，才能进入正常的销售推荐池。
*   **收藏与加购**：目标**50个以上**。虽然没有硬性规定，但数据越高，系统对商品的评分越高，后期推广效果越好。
*   **销量**：
    *   **大类目**：需要**20个以上**的基础销量才能启动自然销售。
    *   **小类目**：需要**10个以上**的基础销量。
    *   **冷门类目**：需根据产品特性和价格具体分析，无统一标准。

## 操作人员与对标分析 🎯

在操作上述基础数据时，不能盲目进行，必须由符合店铺标签的“操作人员”来完成，以确保数据能精准提升店铺权重。

操作人员准备的核心是对目标店铺进行“对标分析”。

以下是进行对标分析时需要关注的维度：

*   **分析对标店铺的产品**：包括产品类型、价格区间、是否具有时效性等。
*   **分析对标店铺的人群**：包括性别、年龄、购买习惯、消费价格敏感度等。

**代码逻辑类比：**
```python
# 对标分析流程
分析对标店铺数据：
    产品属性 = [类型， 价格， 时效性]
    人群画像 = [性别， 年龄， 购买习惯， 消费水平]

根据分析结果，配置操作人员账号：
    操作人员账号特征 ≈ 对标店铺人群画像
    执行基础数据操作（点赞、收藏、加购、购买）
```
通过这种方式，我们让“看起来像目标客户”的账号来为店铺做初始数据，从而为店铺打上正确的人群标签。这个标签在前期确立后，后期很难更改。

![](img/79e86260c9b88217ada0a641fd8af643_7.png)

**重要原则**：店铺前期应**少参与**平台的大型引流活动。在人群标签未稳固前，大型活动引入的杂乱流量会打乱店铺标签，导致活动结束后店铺失去精准流量，难以恢复。前期规划好人群，中后期参与活动则影响不大。
