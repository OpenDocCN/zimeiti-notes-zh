## 用户标签判定逻辑

系统为用户打标签的主要依据是用户的**浏览行为**。用户刚注册时选择的兴趣范围，参考价值不大，它只是平台做的初步内容方向引导。这个初步筛选还无法进入系统最深层的用户标签池。

以下是系统分析用户行为的主要维度：

1.  **点击率**
    *   **公式**：`点击率 = (点击某类内容的次数) / (系统推送的该类内容总次数)`
    *   系统在发现页向用户推送多种内容（如美妆、家居、母婴）。用户点击哪类内容更多，系统就能计算出其偏好。例如，用户点击了10条推送，其中美妆3次，家居2次，母婴0次，系统会判定该用户对母婴内容不感兴趣，并持续推送更多美妆内容进行试探。

2.  **停留时长**
    *   用户点开笔记后的观看时长是关键指标。如果用户频繁点击美妆内容但很快划走，说明这些具体笔记不吸引他。系统会尝试推送美妆领域下的其他细分内容（如护肤、上妆）来观察。反之，如果用户偶然点开一篇家居笔记并停留很久，仔细阅读图片和评论，系统可能判断他有装修或购买家居用品的潜在需求。

3.  **页面浏览量（PV）**
    *   这指的是用户浏览的页面深度。例如，用户不仅看了某篇笔记，还点进博主主页，并连续看了第二、第三篇笔记。提升用户在自己主页的浏览深度，能显著增加账号的健康度评分，对初期没有大量粉丝的账号尤其有益。

4.  **互动行为**
    *   包括点赞、收藏、评论、关注。这些数据直接提升笔记进入下一级推荐流量池的几率。数据好的笔记不仅能获得更多推荐，还可能带动账号内其他笔记的流量，并增加粉丝数，是实现接单、变现等目标的基础。

## 平台流量分发入口

以下是小红书主要的四个流量分发入口：

1.  **发现页**
    *   这是最主要的流量来源。内容包含：
        *   已关注达人发布的内容。
        *   系统根据用户标签（点击率、停留时长等）推荐的、其可能感兴趣的内容。
        *   少量系统试探性推荐的、多样化的内容，以保持平台内容生态的丰富性。

2.  **关注页**
    *   这里集中展示用户所有关注博主的最新笔记。当用户关注列表很长时，可能会在这里寻找特定博主的内容。

3.  **附近页**
    *   基于手机定位，推送同城或附近的内容。此入口以具有强地域特征的内容为主，如美食、本地旅游、线下店铺（美容院、咖啡馆）等。做线下实体店推广的博主，需要重点关注这个入口。

4.  **购物页**
    *   这是小红书向电商发展的重要入口，直接匹配用户的购物需求。用户可在此搜索商品，但许多用户仍习惯先通过搜索笔记来“种草”。笔记中广告过多是目前常见的用户反馈。
