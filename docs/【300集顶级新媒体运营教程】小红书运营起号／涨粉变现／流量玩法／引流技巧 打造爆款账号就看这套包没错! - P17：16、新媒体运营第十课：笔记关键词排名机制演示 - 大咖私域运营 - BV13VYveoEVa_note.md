上一节我们介绍了影响关键词排名的核心因素。本节中，我们来看看一篇笔记从发布到出现在搜索结果中的完整流程。

## 第一步：内容审核

笔记发布后，系统首先会进行内容审核。审核主要关注以下几点：

![](img/294db967315010f5f67d5874c4c94155_11.png)

*   **违规内容**：检查笔记中是否包含敏感词、平台限制准入的类目或其他非法信息。
*   **原创性**：检查笔记是否存在抄袭、素材重复或非原创内容。

内容审核是笔记能否进入后续流程的第一道门槛。

## 第二步：笔记收录

当内容审核通过后，笔记进入收录环节。笔记是否被收录，主要由两个因素决定：

*   **账号权重**：账号在系统流量层级中的位置，权重越高，通常越有利。
*   **笔记质量**：这是更关键的因素。笔记质量由**用户行为数据**决定，例如点击率、互动率等。

![](img/294db967315010f5f67d5874c4c94155_13.png)

**公式示例**：
`笔记收录概率 = f(账号权重， 笔记质量(用户行为数据))`

![](img/294db967315010f5f67d5874c4c94155_15.png)

![](img/294db967315010f5f67d5874c4c94155_17.png)

即使账号权重高，如果新笔记发布后获得的初始曝光（例如1万次）中，点击率极低（例如不足10%），系统也会判定该笔记质量不佳，可能不予收录。反之，一个新账号的笔记如果能在少量曝光中获得较高的互动比例，也可能被成功收录。

## 第三步：特征提取与打标签

笔记被收录的同时，系统会对其内容进行分析，提取特征并打上标签。这个过程决定了你的笔记会被归类到哪些关键词下。

系统主要从以下几个方面提取标签：

1.  **笔记内容关键词**：从标题、正文中自动拆分和提炼出的核心词汇。
2.  **Tag标签**：创作者手动添加的话题标签。
3.  **账号历史数据**：账号以往发布内容被系统打上的标签。
4.  **用户反馈**：笔记分发给初始用户后，这些用户的兴趣标签（例如，对旅游感兴趣的用户点击了你的笔记，系统会推测你的笔记与“旅游”相关）。

**代码逻辑示意**：
```python
# 伪代码：系统提取笔记标签
def extract_note_tags(note_content, manual_tags, account_history_tags, user_feedback_tags):
    # 1. 从内容中提取关键词
    content_keywords = analyze_keywords(note_content)
    # 2. 综合所有来源的标签
    all_potential_tags = combine(content_keywords, manual_tags, account_history_tags, user_feedback_tags)
    # 3. 去重、加权，得出最终标签集合
    final_tags = calculate_final_tags(all_potential_tags)
    return final_tags
```

最终，你的笔记会被打上如“云南”、“旅游”、“学生党”、“避坑”等一系列标签。

## 第四步：内容索引与动态排名

这是核心环节。系统会根据提取的标签，在其庞大的内容索引库中，为你的笔记在各个相关关键词下预先分配一个初始排名。

![](img/294db967315010f5f67d5874c4c94155_19.png)

**重要概念**：排名不是在用户搜索时才临时计算的，而是在笔记收录后就已经在索引库中生成和排序。

![](img/294db967315010f5f67d5874c4c94155_21.png)

### 初始排名与动态调整

我们用一篇假设的笔记《8月去云南旅游的完整攻略，学生党必看避坑指南》来演示：

![](img/294db967315010f5f67d5874c4c94155_23.png)

![](img/294db967315010f5f67d5874c4c94155_25.png)

1.  **初始排名**：系统根据标签，将笔记放入不同关键词的“队列”中，并给予一个初始位置。
    *   在“云南”关键词下，可能排第7位（G）。
    *   在“旅游”关键词下，可能排第6位（F）。
    *   在“云南旅游”关键词下，可能排第4位（D）。
    *   在“学生党”、“避坑”等关键词下，也可能有靠前或靠后的排名。

![](img/294db967315010f5f67d5874c4c94155_27.png)

![](img/294db967315010f5f67d5874c4c94155_29.png)

2.  **动态调整**：初始排名并非一成不变。系统会根据后续真实的**用户搜索行为数据**（如上节课提到的点击率、搜索跳出率、互动数据等）进行动态调整。
    *   **排名上升**：如果在“云南旅游”下的点击率和互动数据很好，笔记在该关键词下的排名可能会从第4位上升到第2位。
    *   **排名下降或移除**：如果在“学生党”这个标签下，长期没有相关搜索用户点击，笔记可能会从这个关键词的排名队列中被移除。同样，“避坑”关键词下的排名也可能因相关性减弱而下降或消失。
    *   **相关性维持**：在“云南”、“旅游”等核心相关但范围较大的关键词下，排名可能会波动，但通常不会被完全移除。

![](img/294db967315010f5f67d5874c4c94155_31.png)

### 动态排名机制

现代新媒体平台比传统搜索引擎更加智能。它们引入了“动态排名”机制。

*   **目的**：为了避免内容同质化，并将流量更合理地分发给不同的创作者。
*   **表现**：对于同一个热门关键词（如“云南旅游”），系统可能维护多个不同的索引库或排序列表。不同用户搜索时，看到的搜索结果排序可能不同。
*   **示例**：用户A搜索“云南旅游”，看到的排名顺序是[笔记B, 笔记A, 笔记C]。而用户B搜索同样的词，看到的顺序可能是[笔记F, 笔记B, 笔记C]。系统根据用户的历史兴趣，动态微调了结果展示顺序。

## 第五步：结果展现

当用户进行搜索时，系统会从已经建立好并经过动态调整的索引库中，提取对应关键词下的笔记排名列表，展示给用户。你的笔记在不同关键词下的最终排名，决定了它能获得多少搜索流量。
