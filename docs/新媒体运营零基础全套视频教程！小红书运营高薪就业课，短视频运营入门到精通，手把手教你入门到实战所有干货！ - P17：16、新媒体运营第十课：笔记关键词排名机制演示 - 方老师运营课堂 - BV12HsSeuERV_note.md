上一节我们介绍了影响搜索排名的核心因素，本节中我们来看看系统是如何运用这些因素，一步步决定笔记排名的。

## 内容审核 🔍

笔记发布后，系统首先会进行内容审核。这一步主要检查笔记是否违规。

![](img/edded4230da628f82fc652a77abef314_11.png)

以下是审核的主要方面：
*   **敏感词与违规内容**：检查笔记中是否包含平台禁止的敏感词汇、限制准入的类目或其他非法信息。
*   **原创性与重复度**：判断笔记内容是否为抄袭，以及素材是否高度重复或非原创。

当内容审核通过后，笔记才会进入下一个环节。

## 笔记收录 ✅

审核过关后，笔记进入收录环节。笔记能否被收录，取决于两个核心因素。

![](img/edded4230da628f82fc652a77abef314_13.png)

![](img/edded4230da628f82fc652a77abef314_15.png)

以下是决定收录的关键：
*   **账号权重**：账号在系统流量层级中的位置，权重越高，基础曝光机会越大。
*   **笔记质量**：由用户的行为数据决定。例如，在一定的初始曝光量下，笔记的点击率（小眼睛数量）是重要指标。即使初始曝光量低，但点击率表现良好，笔记被收录的可能性也会增加。

![](img/edded4230da628f82fc652a77abef314_17.png)

当笔记被成功收录的同时，系统会开始对其内容进行分析。

## 特征提取与打标签 🏷️

系统会对收录的笔记内容进行分析，提取特征并为其打上标签。这决定了笔记会被归类到哪些关键词下。

标签主要通过以下信息提取：
*   **用户标签**：对笔记产生互动（如点击、点赞）的用户身上带有的兴趣标签。
*   **账号历史数据**：账号以往发布内容被系统打上的标签。
*   **内容关键词**：从笔记的标题、正文中直接拆分和提炼出的核心词汇。

例如，一篇关于“云南旅游”的笔记，可能被系统打上 **`云南`**、**`旅游`**、**`云南旅游`**、**`学生党`**、**`避坑`** 等标签。系统会根据这些标签，将笔记放入对应的“关键词库”中，并为它设定一个初始排名。

## 内容索引与动态排名 📈

内容索引是指系统根据提取的标签，在各个关键词的数据库中对笔记进行排序。**这不是在用户搜索时才临时排序，而是在笔记收录后就已经预先排好。**

![](img/edded4230da628f82fc652a77abef314_19.png)

初始排名会根据笔记的标签和早期数据表现设定。但排名并非一成不变，它会根据后续的用户行为数据进行**动态调整**。

![](img/edded4230da628f82fc652a77abef314_21.png)

![](img/edded4230da628f82fc652a77abef314_23.png)

我们可以通过一个例子来理解这个过程。假设一篇标题为 **`《8月去云南旅游的完整攻略，学生党必看避坑指南》`** 的笔记被收录。

系统可能将其放入以下几个关键词库并给予初始排名：
*   **`云南`** 关键词库：排名第7位（假设用“G笔记”表示）。
*   **`旅游`** 关键词库：排名第6位（“F笔记”）。
*   **`云南旅游`** 关键词库：排名第4位（“D笔记”）。
*   **`学生党`** 关键词库：排名第8位（“H笔记”）。
*   **`避坑`** 关键词库：排名第5位（“E笔记”）。

![](img/edded4230da628f82fc652a77abef314_25.png)

![](img/edded4230da628f82fc652a77abef314_27.png)

**排名的动态变化遵循以下规律：**
1.  **数据驱动升降**：如果笔记在 **`云南旅游`** 下的点击率、完播率等数据很好，其在该关键词下的排名可能会上升（例如从第4名升到第2名）。
2.  **剔除不相关标签**：如果笔记在 **`学生党`** 标签下长期没有获得有效点击（因为内容核心是旅游，并非学生话题），系统后期可能会将笔记从这个关键词库中移除，不再参与该词排名。
3.  **弱相关词排名可能下降**：在 **`旅游`** 这样的大词下，由于内容更垂直（专注于云南），排名可能会缓慢下降，但不会完全被移除，因为仍有一定相关性。

![](img/edded4230da628f82fc652a77abef314_29.png)

![](img/edded4230da628f82fc652a77abef314_31.png)

此外，新媒体平台还存在 **“动态排名”** 机制。平台为了将流量更公平地分发给不同博主，并更好地匹配用户的即时兴趣，**同一个关键词的搜索结果可能不会对所有用户完全固定**。

例如：
*   用户A搜索 **`云南旅游`**，看到的排名顺序可能是：笔记1 -> 笔记B -> 笔记C -> 你的笔记…
*   用户B在同时搜索 **`云南旅游`**，看到的排名顺序可能变为：笔记X -> 笔记Y -> 笔记C -> 你的笔记…

这意味着系统可能维护着多个动态的索引库，根据实时数据和对用户偏好的预测，微调展示结果，从而使排名更具流动性和公平性。

## 搜索与展现 🔎

当用户在前端搜索某个关键词时，系统会直接调用已经过动态排序的对应索引库，将排名靠前的笔记展示给用户。你的笔记在哪些关键词下有排名、排名高低，就决定了它能从搜索中获得多少曝光。

![](img/edded4230da628f82fc652a77abef314_33.png)

本节课中我们一起学习了新媒体平台搜索排名的完整机制。我们从**内容审核**开始，经历了**笔记收录**、**特征提取与打标签**，深入到**内容索引与动态排名**的核心过程，最后看到了结果如何通过**搜索与展现**抵达用户。理解这个流程，将帮助你更有针对性地优化笔记，提升在关键词搜索下的排名，从而获得更精准、更持续的流量。
