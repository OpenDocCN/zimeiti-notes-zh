## 概述

小红书的推荐机制与抖音、头条等平台逻辑相似。其核心是基于**用户与内容的标签**进行匹配，并通过**多级流量池**算法逐步分发内容。笔记的传播效果取决于其在每一级流量池中的用户互动数据。

## 内容审核与标签系统

上一节我们介绍了内容创作的基础，本节中我们来看看笔记发布后系统如何处理。所有用户和内容都会被系统打上相应的标签。

当发布一篇笔记时，系统首先会进行**机器审核**，检测内容是否违规。检测会涉及之前课程提到的敏感词汇等内容。

```python
# 伪代码示例：机器审核流程
if 笔记内容.contains(违规词汇):
    进入人工审核流程
else:
    进入算法匹配分发流程
```

如果机器审核无法确定是否违规，笔记会被提交给**人工审核**。只有通过审核的笔记才能进入后续的推荐分发流程。

若笔记被机器和人工均判定为违规，账号可能会受到限制，例如被关进“小黑屋”或限制发言。

此外，系统会提取笔记中的**关键词**、**图片标签**和**话题**等信息，将其分类到对应的内容类目中。用户在注册时选择的兴趣领域，也是为了给自己打上兴趣标签，以便系统推荐相关内容。

## 核心：多级流量池推荐机制

很多人觉得小红书的推荐机制难以捉摸，一篇笔记的阅读量可能忽高忽低。这其实源于其**阶梯式**的流量池推荐模型。

为了便于理解，我们用以下流程来说明：

![](img/ffa866f5c9a81d5377097eadaea2c832_1.png)

1.  **初始流量池**：笔记发布后，系统会将其分配到一个**基础流量池**。这个池子的曝光量通常不大，可能只有一两百或两三百次推荐。
2.  **数据评估**：系统会监控该流量池内用户的**互动参数**，主要包括：**点赞、收藏、转发**。
3.  **流量升级**：如果笔记的互动数据（即上述参数）达到进入下一级流量池的**阈值**，系统就会将其推荐到更大的**二级流量池**（例如1000次曝光）。
4.  **循环放大**：在二级流量池中，系统继续评估互动数据。如果数据依然优秀，达到新的阈值，笔记将被推送到**三级流量池**（例如5000次曝光），以此类推。

![](img/ffa866f5c9a81d5377097eadaea2c832_4.png)

这个过程可以抽象为以下公式：

**笔记能否进入下一级流量池 = f(当前流量池内的点赞率， 收藏率， 转发率) > 阈值**

其中，`f` 代表平台综合计算互动数据的函数。

因此，笔记的流量是**逐步积累**的，不存在发布后立即获得海量曝光的情况。笔记会在一级级流量池中“闯关”，直到其互动数据无法达到进入下一级池子的要求，推荐便会逐渐停止。
