上一节我们介绍了小红书的基本规则框架，本节中我们来看看平台是如何决定将内容推送给哪些用户的。

## 推流规则概述

小红书的推流规则，是指平台根据账号权重、内容标签等多种因素，决定将笔记展示给哪些用户看的系统机制。无论是店铺账号还是个人博主，其获得的流量都与这套规则紧密相关。

## 核心推流逻辑：三重标签匹配

平台的推流主要基于“发现页流量”。系统并非随机展示内容，而是通过一套精密的算法进行筛选和排序。这套算法的核心是**三重标签匹配**。

以下是构成推流基础的三个关键标签：

1.  **账号标签**
    *   **含义**：这是你账号的“身份档案”。系统通过你注册时填写的**兴趣爱好**、个人资料设置、以及历史浏览、互动行为来判定。
    *   **作用**：决定了平台对你账号属性的初步认知。例如，一个经常浏览和收藏美妆教程的账号，会被打上“美妆爱好者”的标签。

2.  **笔记标签**
    *   **含义**：这是单篇笔记的“内容指纹”。系统通过分析笔记的**文案、图片和视频内容**，来判断这篇笔记适合什么样的人群。
    *   **作用**：决定了笔记的目标受众。例如，一篇关于“30岁抗初老护肤”的笔记，会被打上“30-35岁”、“护肤”、“抗衰老”等笔记标签。
    *   **重要区别**：
        *   **产品笔记**：直接带有商品链接的笔记，通常主要分发给**主动搜索相关关键词**的用户。
        *   **引流笔记**：不直接带商品，通过优质内容吸引用户，再引导其搜索购买。这类笔记有机会获得更广泛的推荐流量。

3.  **关键词与话题标签**
    *   **含义**：这是笔记的“搜索关键词”。主要指你在笔记**标题、正文和话题（#）** 中反复提及的核心词汇。
    *   **作用**：当用户搜索或浏览过这些关键词时，系统会将你的笔记匹配给他们。例如，笔记中多次出现“儿童早教”，那么搜索过该词的用户就更可能看到你的笔记。

**公式表示核心逻辑为：**
`精准推流 = 账号标签匹配 + 笔记标签匹配 + 关键词匹配`

平台通过这三重标签的综合判定，将你的笔记推送给最可能对其感兴趣的“小部分人”进行测试。

## 流量层级与增长逻辑

初始推送后，流量的增长并非一蹴而就，而是遵循一个基于数据反馈的层级增长逻辑。

1.  **初始测试**：笔记发布后，系统根据上述三重标签，将其推送给一小批精准用户（例如2000-3000人）。这个初始流量池的大小取决于你的**账号权重**。
2.  **数据考核**：平台会密切关注这批用户对笔记的**互动数据**，包括：点击率、点赞、收藏、评论、转发。
3.  **流量放大**：如果互动数据表现优异（例如点击率和互动率很高），系统会判定该笔记内容优质，受目标用户欢迎。随后，平台会将笔记推送给**标签相似但范围更广**的下一批用户（例如1万-2万人）。
4.  **逐级扩散**：此过程可能重复多次，表现持续好的笔记会进入更大的流量池，获得爆发式曝光。反之，如果初始数据差，流量推送则会很快停止。

**简单来说，流程如下：**
```
精准标签推送 -> 考核互动数据 -> 数据好则扩大推送 -> 进入更高流量层级
```

## 标签的重要性与修正

前期打好标签是成功的基石。如果标签混乱，会导致推送人群不精准，即使有曝光，点击和互动也会很低，笔记自然无法获得后续流量。

*   **标签修正耗时**：如果账号初期标签设置错误（例如想做好妆但被打上了服装标签），修正需要时间和持续努力。
    *   **微调小标签**：约需**7天**。通过主动搜索、浏览、互动目标领域内容来逐渐影响系统判断。
    *   **彻底更改大标签**：可能需要**15天或更久**。需要更长时间、更一致的行为来覆盖旧的用户画像。
*   **“养号”的本质**：所谓前期“养号”，核心目的之一就是通过模拟目标用户的行为（浏览、点赞、收藏同领域内容），为账号打下正确的**初始标签**。
