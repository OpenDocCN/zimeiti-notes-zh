上一节我们介绍了影响搜索排名的核心因素。本节中，我们来看看一篇笔记从发布到获得搜索排名的完整系统流程。

## 第一步：内容审核 🔍

笔记发布后，系统首先会进行内容审核。此步骤主要检查笔记是否存在违规行为。

![](img/5b8ef6fc53f73306ebff8181e87d70c6_9.png)

以下是审核的主要方面：
*   **敏感词与违禁内容**：检查文案中是否包含平台禁止的敏感词汇、限制准入的类目或其他非法信息。
*   **原创性与重复度**：判断内容是否为抄袭，或与平台内现有素材高度重复。非原创内容将在此环节被识别。

内容审核是笔记进入流量池的第一道门槛，确保平台内容生态的健康。

## 第二步：笔记收录 ✅

当内容审核通过后，笔记进入收录环节。笔记能否被收录，取决于两个核心因素：**账号权重**与**笔记质量**。

*   **账号权重**：由账号在系统内的历史表现和当前所处的流量层级决定。权重越高，通常基础曝光量越大。
*   **笔记质量**：由**用户行为数据**决定。即使账号权重高，如果新笔记的点击率（CTR）等互动数据差，也可能不被收录。反之，低权重账号的笔记若初始互动数据良好，被收录的可能性也会增加。

![](img/5b8ef6fc53f73306ebff8181e87d70c6_11.png)

简单来说，系统通过初始曝光下的用户反馈（如点击量、互动率）来初步判断笔记质量，并决定是否将其纳入搜索数据库。

![](img/5b8ef6fc53f73306ebff8181e87d70c6_13.png)

## 第三步：特征提取与打标签 🏷️

笔记被收录的同时，系统会对其内容进行智能分析，提取关键特征并为笔记打上标签。

标签来源主要包括：
*   **内容关键词**：系统从标题、正文中自动拆分和提炼出的核心词汇。
*   **Tag标签**：创作者手动添加的话题标签。
*   **账号历史数据**：账号以往发布内容积累的领域标签。
*   **用户反馈数据**：笔记初始曝光后，互动用户的兴趣标签。例如，一篇旅游笔记若更受“旅游兴趣”用户欢迎，系统会强化其“旅游”相关标签。

这个过程可以概括为：系统通过多维度信息，判断笔记归属的领域或话题，并为其打上相应的`标签`。

## 第四步：内容索引与初始排名 📑

系统根据提取的标签，将笔记放入对应的“关键词索引库”中，并赋予一个初始排名。**这不是实时计算，而是预先排序。**

为了更直观地理解，我们以一篇假设的笔记为例进行演示：

> **笔记标题**：`《8月去云南旅游的完整攻略，学生党必看避坑指南》`

系统可能从中提取出`云南`、`旅游`、`云南旅游`、`学生`、`避坑`等标签。随后，笔记会被分别放入这些标签对应的索引库中，并各有一个初始排名。

![](img/5b8ef6fc53f73306ebff8181e87d70c6_15.png)

![](img/5b8ef6fc53f73306ebff8181e87d70c6_17.png)

我们用以下伪代码结构模拟索引库：
```python
# 假设的索引库结构（关键词：笔记排名列表）
索引库 = {
    “云南”: [笔记A, 笔记B, 笔记C, **我们的笔记**, 笔记E, ...],
    “旅游”: [笔记A, 笔记F, **我们的笔记**, 笔记H, ...],
    “云南旅游”: [笔记A, 笔记B, 笔记C, **我们的笔记**, ...],
    “学生”: [笔记X, **我们的笔记**, 笔记Y, ...], # 初期可能因关键词出现而入选
    “避坑”: [**我们的笔记**, 笔记M, 笔记N, ...], # 初期可能排名靠前
}
```
此时，当用户搜索“云南旅游”时，系统会直接调取“云南旅游”索引库中预设的排名列表进行展示。

## 第五步：动态排名调整 🔄

初始排名并非一成不变。系统会根据笔记在**搜索场景下的持续数据表现**进行动态调整。

![](img/5b8ef6fc53f73306ebff8181e87d70c6_19.png)

![](img/5b8ef6fc53f73306ebff8181e87d70c6_21.png)

![](img/5b8ef6fc53f73306ebff8181e87d70c6_23.png)

调整依据主要包括上一节提到的：**搜索点击率、搜索跳出率、互动数据（点赞、收藏、评论）** 等。

![](img/5b8ef6fc53f73306ebff8181e87d70c6_25.png)

![](img/5b8ef6fc53f73306ebff8181e87d70c6_27.png)

*   **排名上升**：如果笔记在“云南旅游”下的搜索点击率和互动数据很好，其在该索引库中的排名可能会上升（例如从第4名升至第2名）。
*   **排名下降或移除**：如果笔记在“学生”标签下长期无人点击（内容不相关），系统会逐渐将其从该索引库中移除，不再参与“学生”关键词的排名。同样，在“旅游”这类大词下，排名也可能因竞争力不足而下滑。

![](img/5b8ef6fc53f73306ebff8181e87d70c6_29.png)

**核心机制**：索引库内的排名是一个动态过程，由用户通过搜索行为产生的数据持续驱动和优化。

## 第六步：个性化展示（动态排名）🎯

现代新媒体平台更加智能，引入了“动态排名”机制。这意味着，**同一个关键词的搜索结果，对不同用户、甚至同一用户在不同时间的展示顺序都可能不同**。

平台这样做是为了：
1.  **打破内容同质化**：避免优质流量被少数笔记垄断，给更多创作者曝光机会。
2.  **提升用户体验**：根据用户的实时兴趣和行为（如近期点击偏好），个性化调整结果排序，更精准地匹配需求。

因此，笔记的最终搜索排名是“基础索引排名”与“个性化动态调整”共同作用的结果。
